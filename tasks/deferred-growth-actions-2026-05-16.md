# Deferred Growth Actions 2026-05-16

Статус: отложка. Ничего из списка не запускать автоматически.

Источник: `docs/lecture-2026-05-15-growth-and-agent-system-for-vpp.md`

## Что отложено

### Доступы и права

- Проверить доступы к MAX-каналу `СИЛА Проекта`.
- Проверить, есть ли доступ к карточкам Яндекс Бизнес и 2GIS.
- Проверить, есть ли доступы Avito/Profi/Яндекс Услуги, если будем использовать эти площадки.
- Проверить, какие права есть в Bitrix24 для чтения, очистки дублей и реактивации старых контактов.

### Источники аудитории

- Найти конкретные Telegram-каналы по коммерческой недвижимости, складам, аренде, ТЦ, fit-out.
- Найти VK-сообщества предпринимателей, арендаторов, коммерческой недвижимости.
- Найти форумы и публичные обсуждения с вопросами по открытию магазинов, офисов, складов.
- Собрать шорт-лист девелоперов, УК, индустриальных парков, складских операторов.
- Собрать шорт-лист ритейла и франчайзи для коммерческих перепланировок и fit-out.

### Юридическая безопасность

- Запретить автоматическую личную рассылку без отдельного решения.
- Перед outreach проверить законность и правила площадки.
- Все письма и сообщения делать только `approved-only`.
- Не использовать закрытые базы и серые источники.

### Контент и прогрев

- Подготовить 5 B2B-постов для MAX: склад/ангар, коммерческая перепланировка, реконструкция, ТЦ/магазин, КП за 24 часа при полных данных.
- Подготовить 3 коротких экспертных разбора под Telegram/VK.
- Подготовить один пост, который ведет не в форму, а к списку исходных данных для КП.
- Сделать отдельные лендинги позже: склад/ангар, коммерческая перепланировка, реконструкция, ТЦ/магазин.

### Аналитика

- Для каждого источника завести `utm_source`, `utm_campaign`, `segment`, `source_id`.
- В Bitrix24 фиксировать не просто лид, а готовность к КП: `data_completeness`, `missing_data`, `kp_ready`.
- Считать не только подписчиков, а стоимость лида с исходными данными.
- После первых тестов считать ROMI по источникам.

### Платные и внешние действия

- Не запускать рекламу без бюджета, оффера, UTM и стоп-правил.
- Не отправлять сообщения через Telegram/VK/MAX без отдельного подтверждения.
- Не менять карточки Яндекс/2GIS без отдельного подтверждения.
- Не делать deploy и Supabase до решения по публичному AI-помощнику или личному кабинету.
