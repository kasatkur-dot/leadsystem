# Следующий обязательный шаг: GEO/AI source-of-truth

Дата фиксации: 2026-05-07.

Статус: документ создан, публикации и внешние изменения не запускались.

## Почему это важно

Брендовый GEO/AI-срез показал:

- `Яндекс` и `ChatGPT` находят `ООО Вектор Плюс-Про` и канал `СИЛА Проекта`;
- `Claude` и `Perplexity` находят плохо или не связывают компанию, сайт, СРО, услуги и канал;
- значит, проблема не в полном отсутствии публичного следа, а в слабой связке источников.

Нужна единая публичная связка:

```text
ООО Вектор Плюс-Про
-> ИНН/ОГРН
-> новый сайт
-> СРО
-> услуги
-> кейсы
-> СИЛА Проекта
-> Яндекс Карты / 2GIS / внешние источники
```

## Что нужно подготовить

Создан документ:

```text
docs/geo-ai-source-of-truth.md
```

В документе описать, что именно должно быть:

- на сайте;
- в Яндекс Картах;
- в описании канала `СИЛА Проекта`;
- в закрепленном посте канала;
- в карточках услуг;
- в кейсах;
- во внешних публикациях;
- в технической индексации.

## Минимальные блоки документа

1. Единое название бренда:

```text
ООО «Вектор Плюс-Про»
ИНН 0100005517
ОГРН 1230100001812
```

2. Официальная связка:

```text
сайт -> Яндекс Карты -> СРО -> канал СИЛА Проекта -> контакты
```

3. Услуги, которые нужно назвать прямо:

- проектирование перепланировок;
- проектная документация;
- рабочая документация;
- `КР`;
- `КЖ`;
- `КМ`;
- реконструкция;
- коммерческие объекты;
- склады, ангары, промышленные здания;
- сопровождение экспертизы;
- авторский надзор.

4. Кейсы:

```text
тип объекта -> город/регион -> задача -> разделы -> стадия -> результат -> доказательство
```

5. Канал `СИЛА Проекта`:

- указать, что канал связан с `ООО Вектор Плюс-Про`;
- добавить ИНН/ОГРН или ссылку на официальный сайт;
- добавить ссылку на Яндекс Карты;
- объяснить темы канала: перепланировки, оформление домов, ПД/РД, `КР/КЖ/КМ`, экспертиза.

## Что нельзя делать без согласования

- Не публиковать изменения на сайт.
- Не менять карточки Яндекс/2GIS без подтверждения.
- Не писать неподтвержденные услуги.
- Не использовать реальные клиентские данные.
- Не запускать платные GEO/SEO-сервисы.
- Не добавлять новых агентов.
- Не менять MVP-архитектуру.

## Кому относится в системе

| Блок | Ответственный |
|---|---|
| Анализ AI-видимости | Agent 1 Scout |
| Упаковка сайта, кейсов, канала | Agent 4 Publisher |

