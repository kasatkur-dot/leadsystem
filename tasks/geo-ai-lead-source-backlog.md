# Backlog: будущий слой GEO/AI lead source

Дата: 2026-05-07

Статус: отложенный backlog после первого MVP. Новые агенты не создаются. MVP-архитектура не меняется.

## 1. Что это за слой

`GEO/AI lead source` — это будущий источник лидов и доверия из AI-ответов, нейропоиска и поисковой выдачи.

Простыми словами:

```text
человек спрашивает ChatGPT/Perplexity/Алису/поиск -> видит компанию или конкурентов -> переходит на сайт/карточку/статью -> оставляет заявку -> Agent 5 считает источник
```

Это не отдельный новый агент. Это будущий источник и аналитический слой внутри уже существующей системы:

- Agent 1 Scout: проверяет AI-ответы, запросы, конкурентов и причины видимости;
- Agent 4 Publisher: усиливает сайт, статьи, кейсы, карточки услуг и контент;
- Agent 5 CRM/Analytics: считает AI-referrer traffic, лиды, сделки, выручку и ROMI.

## 2. Какие AI-ответы и нейропоиск могут давать лиды

Потенциальные точки первого касания:

| Источник | Как может дать лид |
|---|---|
| ChatGPT с web/browsing | Пользователь спрашивает, кого выбрать, и получает список компаний/ссылок |
| Perplexity | Пользователь получает ответ со ссылками на сайты, статьи, рейтинги и карточки |
| Яндекс/Алиса | Пользователь получает локальные и поисковые рекомендации |
| Google AI Overviews / нейроответы | Пользователь видит обобщенный ответ и источники |
| Яндекс поиск + нейроответ | Пользователь видит статьи, карточки, сайты и справочники |
| AI-поиск по картам/отзывам | Пользователь выбирает по рейтингу, отзывам, адресу и фото |
| Внешние рейтинги и статьи | AI использует их как доказательства и приводит компанию в ответе |

Важно: этот слой работает только если у компании есть понятные публичные доказательства.

## 3. Как позже отслеживать AI-referrer traffic

После первого MVP и появления сайта/аналитики нужно добавить отдельный сегмент:

```text
source_category = ai_referrer
```

Минимальные поля для аналитики:

| Поле | Зачем |
|---|---|
| `visitor_referrer` | Откуда пришёл переход |
| `landing_page` | На какую страницу пришёл человек |
| `utm_source` | Если можно проставить вручную |
| `utm_medium` | Например `ai`, `geo`, `referral`, `organic` |
| `utm_campaign` | Какой аудит/кампания/страница |
| `query_cluster` | Группа запроса: ипотека, СРО, проект, коммерция |
| `ai_platform` | ChatGPT, Perplexity, Yandex/Alice, Google AI, other |
| `lead_id` | Связь с лидом |
| `deal_id` | Связь со сделкой |
| `revenue_rub` | Выручка |
| `romi` | Окупаемость источника |

Что отслеживать в referrer/domain позже:

- `chatgpt.com`;
- `perplexity.ai`;
- `yandex.ru` / `alice`-связанные переходы, если видны в аналитике;
- `google.com` с AI/organic сегментацией, если возможно;
- другие AI-поисковики после ручной проверки.

Важно: точный regex не фиксируем сейчас. Его нужно составить позже по актуальным доменам и тестовым переходам.

## 6. Какие данные нужны для диагностики “почему нас не упомянули”

Если `Вектор Плюс-Про` не упомянут, нужно собрать не догадки, а факты.

Диагностические данные:

| Данные | Что смотрим |
|---|---|
| Сайт | Есть ли индексируемые страницы под конкретные услуги |
| Страницы услуг | Понятны ли услуга, регион, документы, процесс, результат |
| Кейсы | Есть ли объект, задача, документы, результат, польза |
| Отзывы | Есть ли публичные отзывы на картах/площадках |
| Карточки | Яндекс Карты, 2GIS, фото, услуги, рейтинг, обновления |
| СРО/компетенция | Как доказана проектная часть |
| Внешние упоминания | СМИ, статьи, каталоги, рейтинги, Дзен, VC, профильные площадки |
| Брендовые запросы | Что AI знает о `Вектор Плюс-Про` и `СИЛА Проекта` |
| Конкурентные факторы | Почему AI выбрал PEREPLAN/Ресог/других |
| Техническая индексация | Видит ли поиск нужные страницы |

## 7. Где подключать в цепочке после Redis/Telegram/Bitrix24 MVP

Текущая MVP-цепочка не меняется:

```text
.env -> Redis -> Telegram-сессии -> Bitrix24 -> один тестовый лид
```

GEO/AI layer подключать только после первого MVP так:

```text
Agent 1 Scout manual GEO audit
-> Agent 4 Publisher улучшает страницы, кейсы, статьи, карточки
-> сайт/карточки/AI-ответы дают переходы
-> Agent 2 или сайт-форма/CRM получает лид
-> Agent 3 обрабатывает лид
-> Agent 5 CRM/Analytics считает ai_referrer, сделку, выручку, ROMI
```

Простыми словами:

- сначала доказываем, что основной путь лида работает;
- потом усиливаем видимость;
- потом смотрим, появились ли переходы и заявки из AI/поиска;
- потом считаем деньги.

## 8. Что сейчас не делаем

- Не создаём новых агентов.
- Не меняем `orchestrator/scheduler.py`.
- Не запускаем массовый аудит AI-ответов.
- Не подключаем платные GEO-сервисы.
- Не публикуем страницы или статьи без согласования.
- Не обещаем пользователям несуществующие услуги.
- Не отправляем клиентские данные во внешние AI-сервисы.

## 9. Когда вернуться к этому backlog

Вернуться после:

1. `.env` обязательные поля закрыты;
2. Redis проверен;
3. Telegram/Bitrix24 MVP проверен на одном тестовом лиде;
4. есть хотя бы один сайт/страница/карточка, куда можно вести AI/referrer traffic;
5. есть ручной `Срез 1-6` в `research/geo-ai-visibility-audit.md`.
