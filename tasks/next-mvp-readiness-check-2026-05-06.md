# Следующая задача: безопасная подготовка MVP-прогона

Дата постановки: 2026-05-06

Статус: готово к запуску в новом чате проекта.

## Зачем задача

Проект уже хорошо обновлен по урокам и многоагентной карте. Сейчас не нужно добавлять новых агентов, новые источники или новые сервисы.

Главная цель следующего шага:

```text
.env -> Redis -> Telegram-сессии -> Bitrix24 -> один сквозной MVP-тест
```

Пока не закрыты `.env`, Redis, Telegram-сессии и Bitrix24, нельзя честно проверять всю систему.

## Что нельзя делать в этой задаче

- Не показывать значения секретов из `.env`.
- Не запускать платные API.
- Не запускать массовый сбор лидов.
- Не публиковать реальные посты.
- Не добавлять новых агентов.
- Не менять архитектуру.
- Не делать commit/push без отдельного подтверждения.

## Что нужно сделать

1. Прочитать стартовые файлы проекта.
2. Проверить `.env` безопасно: показывать только `SET` или `EMPTY`.
3. Сравнить `.env` с `.env.example`.
4. Выделить обязательные поля MVP.
5. Отдельно выделить поля для MAX/Agent 4 publisher.
6. Если обязательные поля пустые, не запускать Redis и систему.
7. Если поля заполнены, проверить Redis без разрушительных действий.
8. Проверить готовность Telegram-сессий по наличию файлов/настроек, не раскрывая данные.
9. Проверить готовность Bitrix24 только по статусу webhook `SET/EMPTY`.
10. В конце дать понятный список: что должна заполнить Яна, что может сделать агент, какой следующий маленький шаг.
11. Обновить `REPORT.md` и `tasks/session-notes.md`.
