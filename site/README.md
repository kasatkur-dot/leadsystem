# Новый сайт Вектор Плюс-Про

Статус: новая локальная версия на базе архива `WPP2.zip` от Claude Design.

## Что внутри

- `index.html` — точка входа сайта.
- `services/*/index.html` — статические индексируемые страницы услуг для SEO/GEO.
- `sections/*/index.html` — статические справочные страницы по разделам ПД/РД: основные разделы, инженерия и рабочие комплекты.
- `src/app.jsx`, `src/sections.jsx`, `src/icons.jsx`, `src/images.js` — React/JSX-структура сайта.
- `assets/app.bundle.js` — собранная версия React/JSX для обычного открытия через `file://` и статического preview.
- `assets/vendor/` — локальные React/ReactDOM, чтобы сайт не зависел от CDN при открытии файла.
- `tools/build-bundle.mjs` — пересборка `assets/app.bundle.js` после изменений в `src/*`.
- `styles.css` — визуальная система новой версии.
- `assets/brand/` — настоящий логотип Вектор Плюс-Про из базы проекта.
- `assets/images/` — локальные WebP-фото-визуалы WPP2 без клиентских данных.
- `assets/media/project-path-demo.mp4` — локальное видео для медиа-блока.
- `robots.txt`, `sitemap.xml` — базовые файлы для будущей публикации.
- `open-site.command` — локальный запуск через Python HTTP server.
- `tools/generate-local-visuals.py` — пересоздание локальных нейтральных визуалов.
- `tools/generate-service-pages.py` — пересоздание статических страниц услуг и `sitemap.xml`.
- `tools/generate-section-guide-pages.py` — пересоздание справочных страниц `sections/*/index.html`.

## Важно

- Сайт пока не опубликован.
- Vercel/GitHub deploy не запускался.
- Формы, CRM, Bitrix24 и внешняя отправка заявок не подключались.
- Актуальное рабочее ТЗ на следующую доработку сайта: `../docs/current-site-improvement-tz.md`.
- Правило связи сайта с Bitrix24 без формы: `../docs/site-bitrix24-no-form-handoff.md`.
- Старый сайт из папки `site` удалён из проекта; старые страницы услуг не используются.
- Новый сайт сейчас работает как статическая React-страница с локальным `assets/app.bundle.js`. Это исправляет пустую страницу при открытии через `file://`.
- Если меняются файлы `src/*`, нужно пересобрать бандл командой из корня проекта: `node site/tools/build-bundle.mjs`.
- Если нужно пересоздать нейтральные схематичные визуалы, команда из корня проекта: `.venv/bin/python site/tools/generate-local-visuals.py`. Сейчас это не основной визуальный стиль, потому что на сайте возвращены фото-визуалы WPP2.
- Если меняется контент статических страниц услуг, команда из корня проекта: `.venv/bin/python site/tools/generate-service-pages.py`.
- Если меняются справочные страницы по разделам, команда из корня проекта: `.venv/bin/python site/tools/generate-section-guide-pages.py`.
- Перед production-публикацией всё равно желательно сделать полноценную сборку/preview без CDN-шрифтов и с финальными локальными визуалами.

## Production-аудит 2026-05-11

Сохранено после проверки сайта как будущей публичной точки входа для лидов:

- `index.html` содержит favicon, apple-touch-icon, базовые SEO/OG/JSON-LD, `referrer=strict-origin-when-cross-origin` и `noscript`-fallback с понятным текстом и контактами.
- React запускается через локальные runtime-файлы и собранный `assets/app.bundle.js`, поэтому сайт открывается через `file://` и локальный preview без Babel в браузере.
- Логотипы в шапке и меню имеют явные размеры, чтобы уменьшить сдвиги макета при загрузке.
- FAQ, модальные окна, мобильное меню и внутренние якоря получили базовые улучшения доступности: `aria-controls`, `role=region`, понятный `aria-label`, `scroll-margin` и `overscroll-behavior`.
- В CSS добавлены правила для `prefers-reduced-motion`, `touch-action`, читаемого hero на desktop/mobile и безопасного отображения без JavaScript.
- Быстрые контакты и подвал должны оставаться кликабельными: телефон, email, MAX-аккаунт, MAX-канал `СИЛА Проекта`, Telegram, Яндекс Карты и 2GIS.

## Перед публикацией

- Проверить, что все изображения лежат локально, сжаты и не содержат клиентских/персональных данных.
- Сделать static/prerender или полноценную production-сборку, чтобы SEO/GEO не зависели только от клиентского JavaScript.
- Проверить desktop/tablet/mobile в реальном браузере, включая переносы H1, CTA, карточки услуг, кейсы, медиа, контакты и подвал.
- Проверить security headers уже на хостинге: `Content-Security-Policy`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`.
- Не подключать формы, CRM, Bitrix24, аналитику и внешнюю отправку заявок без отдельного подтверждения.

## Правила

- Не обещать результат согласования перепланировок до проверки объекта, документов и требований органов.
- Не публиковать клиентские адреса, ФИО, телефоны, суммы, договоры, кадастровые номера, штампы и внутренние документы.
- Контакты должны быть кликабельными: телефон, email, MAX-аккаунт, MAX-канал, Telegram, Яндекс Карты, 2GIS.
- Медиа не должно запускать звук автоматически.
- Перед публикацией проверить desktop/tablet/mobile, все ссылки, SEO/OG/JSON-LD и доступность.
