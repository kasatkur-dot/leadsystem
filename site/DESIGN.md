# DESIGN.md — WPP2 версия сайта

## Направление

Новая версия сайта построена как светлый архитектурно-инженерный лендинг:

- тёплый бумажный фон;
- стеклянные карточки;
- крупные строительные/архитектурные визуалы;
- акцент на проектной организации, СРО, документации, реконструкциях и перепланировках;
- больше экспертного текста для клиентов и GEO/SEO-видимости.

## Что взято из старой версии как полезная технология

- локальный запуск `open-site.command`;
- настоящий логотип из базы проекта;
- локальный медиа-ассет `project-path-demo.mp4`;
- правила безопасных контактов, SEO/GEO и запрет на рискованные обещания;
- `robots.txt` и `sitemap.xml`.

## Что удалено

- старый `index.html`;
- старые CSS/JS;
- старые SVG/постерные кейс-картинки;
- старые страницы услуг `services/*`.

## Перед следующей итерацией

- Проверить визуально первый экран, услуги, кейсы, медиа, контакты и мобильную версию.
- Заменить React+Babel CDN на production-сборку, если сайт пойдёт на публичный домен.
- Если понадобятся отдельные SEO-страницы услуг, создавать их уже в новом стиле WPP2, а не возвращать старые страницы.

## Визуалы

- На сайте используются локальные фото-визуалы WPP2 в `assets/images/*.webp`.
- Эти визуалы не должны тянуться с внешних URL на публичной версии: картинки должны лежать локально и открываться без зависимости от сторонних сервисов.
- Для финальной публикации желательно либо подтвердить право использования выбранных фото, либо заменить их на собственные/сгенерированные визуалы в том же современном архитектурном стиле.
- Нельзя публиковать реальные проектные скрины без обезличивания адресов, ФИО, телефонов, сумм, договоров, кадастровых номеров, штампов и служебных отметок.

## Дополнение по доверию

- На сайте должен быть виден не только бренд компании, но и состав проектной ответственности: директор, ГИП и профильные специалисты.
- Имена публикуются только по подтверждённым данным. Сейчас явно указаны:
  - директор: Черная Яна Александровна;
  - ГИП: Макеев Сергей Владимирович.
- Остальные участники показываются как роли: архитектор, конструктор/КР, инженер КЖ, инженер КМ, специалист по исходным данным, координатор документации.
- Не писать на сайте дату открытия компании как отдельное преимущество; в подвале использовать период `2020–2026`.

## Production-правила после аудита 2026-05-11

- Сайт должен оцениваться как полноценный публичный продукт, а не как MVP: первый экран, услуги, кейсы, доверие, команда, FAQ, контакты и подвал должны выглядеть завершённо.
- Визуальный стиль: светлая архитектурно-инженерная палитра, ясный контраст, аккуратные акценты, меньше мелких плашек на первом экране, больше воздуха и понятных блоков.
- Hero должен быстро отвечать на вопрос клиента: кто мы, какие задачи закрываем, что получает заказчик и как сделать первый шаг.
- В текстах держать баланс SEO/GEO и человеческого языка: проектная документация, рабочая документация, КР, КЖ, КМ, перепланировки, реконструкции, РНС, Краснодар, работа по России.
- Нельзя обещать гарантированное согласование, разрешение или результат до проверки объекта, исходных данных и требований уполномоченных органов.
- Доверие строится через проверяемые факты: ИНН, ОГРН, СРО, команда проекта, безопасные кейсы, карты, контакты и связь с MAX-каналом `СИЛА Проекта`.
- Реальные проектные скрины можно добавлять только после обезличивания адресов, ФИО, телефонов, сумм, договоров, кадастровых номеров, штампов и служебных отметок.
- Визуалы должны быть локальными и оптимизированными; внешние картинки допустимы только для чернового просмотра, не для финальной публикации.
- Для accessibility сохранять видимый focus, понятные aria-label, нормальный контраст, корректные alt и отсутствие автозвука.
- Для performance сохранять явные размеры логотипов/изображений, уменьшать вес первого экрана, учитывать reduced motion и не использовать тяжёлые внешние зависимости без необходимости.
- Для production нужна статическая/prerender-версия или сборка, где важный контент доступен поиску и AI-ассистентам без ожидания клиентского JavaScript.

## Что ещё не считать финальным 10/10

- Нет финальной проверки на реальных устройствах: desktop, tablet, mobile.
- Нет публичного хостинга с проверенными security headers.
- Нет отдельных SEO-страниц услуг в новом стиле.
- Нет окончательно утверждённых локальных визуалов по всем блокам.
- Нет подключённой аналитики лидов через Agent 5, потому что формы/CRM пока сознательно не подключались.
