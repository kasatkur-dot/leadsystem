# Content Navigation

Эта папка хранит контентную систему проекта `design-studio-lead-engine`: идеи, черновики постов, визуальные промпты, планы публикаций, источники, брендовые материалы и безопасные кейсы для сайта.

Перед любой работой с контентом сначала читать:

1. `../REPORT.md` — последняя точка сверки проекта.
2. `pipeline/published-log.md` — единственный источник правды о том, что уже опубликовано.
3. `pipeline/monthly-content-plan-2026-05.md` — текущий месячный план.
4. `docs/content-approval-cycle.md` — правила согласования контента.

## Структура

- `library/posts/` — библиотека текстов постов. Название `draft-*` означает рабочий файл, а не статус публикации.
- `library/visual_prompts/` — промпты для визуалов к идеям и постам.
- `library/templates/` — шаблоны карточек и видео.
- `library/sources/` — реестры источников, каналов, конкурентов и идей.
- `pipeline/` — контент-план, журнал публикаций, dry-run и ретроспектива.
- `strategy/` — стратегия канала, аудитория и сценарий входа по слову `ПРОВЕРКА`.
- `studio_brand/` — голос студии, страхи аудитории и профили клиентов.
- `site/` — только безопасные обезличенные материалы для сайта.
- `bot_spec/` — ТЗ для MAX-бота и сценариев канала.

## Правила публикации

- Не считать файл опубликованным по названию. Проверять только `pipeline/published-log.md`.
- Не публиковать и не планировать повторно рядом по времени то, что уже есть в `published-log.md`.
- До публикации статус должен пройти цепочку:

```text
planned -> draft -> review -> needs_changes / approved
```

- Статусы `scheduled -> published -> analyzed` использовать только после отдельного разрешения на реальную публикацию.
- Автопубликация, PostMyPost, CRM-интеграции и внешние рассылки не запускаются без отдельного подтверждения.

## Безопасность

- Не вставлять сюда реальные токены, пароли, cookie, закрытые переписки, договоры, счета, адреса объектов, ФИО клиентов, телефоны клиентов и кадастровые номера.
- Для сайта использовать только `site/cases-safe-mvp.md` и материалы, прошедшие обезличивание.
- Конкурентов и внешние каналы использовать как источник паттернов, а не как текст для копирования.
