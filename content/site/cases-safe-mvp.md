# Безопасные кейсы для сайта Вектор Плюс-Про

Дата: 2026-05-07.

Статус: подготовлены публичные обезличенные карточки для первой версии сайта. На сайт выносятся только сведения, которые помогают оценить компетенцию компании: тип объекта, задача, состав работ, разделы и результат. Точные источники, имена заказчиков, адреса, телефоны, договоры, суммы, платежи, штампы чертежей, сканы документов и внутренние переписки не публикуются.

## Правило публикации

На сайте можно показывать:

- тип объекта;
- общий регион или географию без точного адреса;
- задачу клиента;
- состав работ;
- нейтральный результат;
- связанные услуги.
- обезличенный фрагмент чертежа или схемы, если на нём убраны штампы, адреса, ФИО, телефоны, номера документов и служебные отметки.

На сайте нельзя показывать без отдельного разрешения:

- ФИО и телефоны заказчиков;
- точные адреса объектов;
- кадастровые номера;
- договоры, счета, чеки, платежные документы;
- спорные переписки и претензии;
- внутренние рабочие файлы;
- точные суммы договоров;
- персональные данные исполнителей и клиентов.
- штампы чертежей, исходные выписки, технические паспорта, кадастровые номера и любые изображения, по которым можно восстановить объект или заказчика.

## Правило визуалов

Для первой версии сайта используем нейтральные визуальные карточки без чужих данных. Реальные скрины из проектов добавлять только после отдельного отбора и обезличивания.

Что найдено при быстрой проверке локальной базы:

- есть реальные фотографии и скрины проектных материалов;
- часть изображений содержит адреса, кадастровые сведения, договорные или личные данные;
- такие материалы нельзя переносить на сайт напрямую;
- для публичной версии нужен отдельный этап: выбрать 3-5 изображений, убрать все идентификаторы, затем заменить нейтральные визуалы в карточках.

Что сделано для текущей версии сайта:

- созданы безопасные SVG-визуалы в `site/assets/cases/`;
- визуалы не используют реальные адреса, заказчиков, кадастровые номера, штампы, суммы и телефоны;
- визуалы можно показывать на сайте как промо-иллюстрации кейсов до тех пор, пока не будет согласован пакет реальных обезличенных скринов.

## Кейсы для первой волны сайта

### 1. Образовательный объект на 550 мест

Публичный тип: социальная инфраструктура.

Задача: подготовить рабочую документацию по проектной части объекта с высокой плотностью инженерных и планировочных требований.

Что сделано: выполнена работа с комплектом проектных материалов по согласованному составу.

Безопасный результат: заказчик получил структурированный комплект материалов для дальнейшей работы по объекту.

Связанные услуги: `РД`, проектная координация, социальные объекты.

Не публиковать: точный адрес, документы, реквизиты договоров, имена сторон.

### 2. Капитальный ремонт здания школы

Публичный тип: объект социальной инфраструктуры.

Задача: разработать конструктивные решения по усилению и восстановлению несущих конструкций, включая расчёт стропильной системы.

Что сделано: подготовлены материалы раздела `КР` и расчётная часть по согласованному составу исходных данных.

Безопасный результат: заказчик получил конструктивную основу для дальнейшей проектной работы по капитальному ремонту.

Связанные услуги: `КР`, расчёты, усиление конструкций, капитальный ремонт.

Не публиковать: точный адрес, номер школы, спорные материалы, претензии, суммы, персональные данные.

### 3. Производственно-складской объект с торговой функцией

Публичный тип: коммерческий / производственно-складской объект.

Задача: определить состав проектной и рабочей документации для нового производственно-складского здания с торговой функцией.

Что сделано: собран перечень разделов для проектной и рабочей стадии: электрика, слаботочные системы, водоснабжение, вентиляция, автоматизация.

Безопасный результат: заказчик получил структурированную карту разделов, сроков и состава работ для проектирования.

Связанные услуги: `ПД`, `РД`, инженерные разделы, коммерческие объекты.

Не публиковать: точный адрес, кадастровый номер, названия контрагентов, коммерческие суммы.

### 4. Перепланировки квартир

Публичный тип: квартиры в многоквартирных домах.

Задача: проверить исходные документы, обмеры и ограничения помещения перед подготовкой проектных материалов.

Что сделано: подготовлены проектные материалы по архитектурной части и сформирован список следующих шагов для работы с уполномоченным органом.

Безопасный результат: заказчик получает проектную основу и понимание, какие документы и этапы нужны дальше.

Связанные услуги: перепланировки, `АС`, `АР`, исходные данные, документы.

Не публиковать: адрес квартиры, ФИО собственника, техпаспорт, выписки, доверенности, платежные документы.

### 5. Металлоконструкции для промышленного объекта

Публичный тип: промышленный объект.

Задача: на основании `КМ` и технической документации подготовить деталировочные чертежи для изготовления и монтажа металлических конструкций.

Что сделано: подготовлены материалы `КМД`: монтажные схемы, отправочные марки, ведомости стали и метизов.

Безопасный результат: заказчик получил комплект деталировочной документации для дальнейшей производственной работы.

Связанные услуги: `КМД`, `КМ`, металлоконструкции, монтаж.

Не публиковать: название промышленного объекта, контрагентов, веса, договорные суммы, внутренние графики.

## Что ещё нужно позже

- Выбрать 3-5 кейсов, которые можно публично подтвердить с разрешением заказчика.
- Подготовить 1-2 изображения или схемы без персональных данных: обезличенный фрагмент чертежа, нейтральный 3D-визуал или сгенерированную иллюстрацию без привязки к реальному заказчику.
- Для каждого кейса добавить мини-CTA: `Похожая задача? Напишите город, тип объекта и исходные данные`.
- После выбора домена связать кейсы с SEO-страницами услуг.
