# Навыки для доработки нового сайта

Дата: 2026-05-07.

Статус: внутренний документ. Сайт не опубликован.

## 1. Что искали

Яника попросила подключить через `find-skills` нужные навыки для современного и привлекательного сайта Вектор Плюс-Про.

## 2. Что получилось

`find-skills` подключён как регламент поиска навыков.

Попытка выполнить `npx skills find "web design ui ux accessibility landing page"` и `npx skills find "website conversion landing page design"` в текущей среде зависла без вывода. Вероятная причина: сетевые ограничения среды.

Чтобы не тормозить работу, используем уже доступные локальные навыки.

## 3. Навыки, которые используем сейчас

### `web-design-guidelines`

Для чего:

- проверить UI/UX;
- проверить доступность;
- проверить hover/focus-состояния;
- не сделать красивый, но неудобный сайт.

Что применяем сразу:

- `skip link`;
- видимые `focus-visible` состояния;
- корректные alt/aria;
- ссылки для навигации вместо кликов по div;
- адаптивность;
- уважение к `prefers-reduced-motion`.

### `browser-use`

Для чего:

- открывать сайт в браузере;
- проверять десктоп/мобильный вид;
- делать визуальную QA-проверку.

Ограничение:

- в песочнице Codex локальный порт может быть недоступен;
- поэтому визуальную проверку иногда нужно делать через локальный запускатель `site/open-site.command`.

### `imagegen`

Для чего позже:

- сгенерировать недостающие фоновые иллюстрации;
- подготовить аккуратные hero/preview-визуалы;
- не трогать клиентские данные.

Важно:

- настоящий логотип Вектор Плюс-Про не генерируем заново, а подключаем из файла Яники.

## 4. Навыки, которые пригодятся позже

### `HyperFrames`

Для чего:

- промо-видео;
- анимация пути клиента;
- короткие ролики для MAX/Telegram/VK/Дзена.

### `Remotion`

Для чего:

- программные видео;
- повторяемые шаблоны роликов;
- ролики на основе сайта и кейсов.

### `vercel-react-best-practices`

Для чего:

- если переводим сайт из статического HTML в React/Next.js;
- оптимизация производительности;
- подготовка к Vercel.

## 5. Что делаем в первой волне

- Улучшаем текущий статический сайт.
- Подключаем настоящий логотип после получения файла.
- Делаем современный первый экран, понятные CTA и аккуратный путь клиента.
- Не подключаем CRM, формы, аналитику и Vercel без отдельного шага.

## 6. Что не делаем сейчас

- Не публикуем сайт.
- Не запускаем платные сервисы.
- Не создаём новых агентов.
- Не генерируем логотип вместо настоящего.
- Не используем клиентские документы и реальные кейсы без ручного подтверждения.
