## 1. Что именно понял из метода лектора

Лектор привлекал подписчиков и заявки не одним способом. Он построил систему:

```text
агенты
-> разведка источников
-> контент-завод
-> продуктовый вход
-> массовое тестирование каналов
-> прогрев
-> заявки
-> аналитика
-> оптимизация бюджета
```

Важные части:

- много источников, а не ставка на один канал;
- контент делается серийно;
- человек попадает не сразу в продажу, а в понятный продуктовый вход;
- дальше система догревает и ведет к заявке;
- всё считается по стоимости и качеству лида;
- опасные действия требуют юридической проверки.

## 2. Как это переносится на ВПП

Для ВПП продуктовый вход не развлекательный, а экспертный:

```text
Проверка готовности исходных данных к КП
```

Это сильнее, чем просто “оставьте заявку”, потому что клиент сразу понимает:

- что ему нужно прислать;
- можно ли получить КП за 24 часа;
- где риск по объекту;
- какой следующий шаг.

## 3. Первая цепочка привлечения

```text
Источник аудитории
-> полезный экспертный материал
-> мягкий переход в проверку исходных данных
-> Bitrix24 с source_id и segment
-> КП или список недостающих данных
-> договор / отказ / отложенный прогрев
```

## 4. Приоритетные сегменты

| Приоритет | Сегмент | Почему |
|---|---|---|
| P1 | коммерческая перепланировка | юрлица, срочные сроки, понятный больной вход |
| P1 | склады и ангары | высокий чек, B2B, нужны ПД/РД и СРО |
| P1 | реконструкция | сложные проекты, нужна экспертиза и опыт |
| P2 | ТЦ, магазины, fit-out | много арендаторов и требований УК/арендодателей |
| P2 | отдельные разделы ПД/РД | можно брать как вход в B2B |
| P3 | частные дома | сохраняем, но не первый growth-фокус |

## 5. Первые источники

| Источник | Что делаем сейчас | Что нельзя без approval |
|---|---|---|
| MAX `СИЛА Проекта` | готовим B2B-посты | публиковать и продвигать |
| Bitrix24 старая база | read-only чистка и сегментация | писать клиентам, merge, update |
| Тендеры | фильтры и решение участвовать/нет | регистрироваться, платить, подавать |
| Яндекс/2GIS | подготовить чеклист профиля | менять карточки |
| Telegram/VK/форумы | собрать список площадок | писать, комментировать, закупать |
| ABM сайты компаний | собрать шорт-лист | отправлять письма |

## 6. Первая метрика

Не “подписчик любой ценой”.

Главная метрика ВПП:

```text
стоимость лида с достаточными исходными данными для КП
```

Дополнительные:

- `source_id`;
- `segment`;
- `question_count`;
- `source_data_received`;
- `kp_ready`;
- `kp_sent`;
- `contract_won`;
- `revenue`;
- `ROMI`.

## 7. Stop rules

Остановиться до отдельного решения, если действие:

- пишет человеку лично;
- публикует пост;
- запускает рекламу;
- меняет Bitrix24;
- объединяет дубли;
- регистрируется на площадке;
- платит за сервис;
- выгружает документы наружу;
- использует спорный источник данных.

1. Подготовить локальный acquisition dry-run по 5 сегментам.
2. Для каждого сегмента сформировать:
   - источник;
   - оффер;
   - пост/лендинг;
   - UTM;
   - Bitrix24-метки;
   - первый ответ AI-менеджера;
   - критерий остановки.
3. После approval выбрать один тест и сделать маленький запуск.
