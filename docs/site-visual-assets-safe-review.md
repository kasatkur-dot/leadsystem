## 2. Решение по логотипам

### `логотип1.png`

Путь:

```text

```

Что внутри:

- горизонтальный логотип `ВЕКТОР ПЛЮС-ПРО`;
- знак с цветными зданиями;
- слоган `СЛОЖНЫЕ ПРОЕКТЫ ПРОСТО`.

Решение:

- можно использовать как временный брендовый ориентир;

- для нового сайта лучше не вставлять как финальный логотип без адаптации.

Почему:

- стиль выглядит более старым и цветным, чем новое направление сайта;
- для современного сайта лучше сделать аккуратный wordmark или обновлённую версию в более спокойной палитре.

Статус:

```text

```

## 5. Что можно брать в макет первой волны

Можно брать как основу:

- `логотип1.png` — как временный ориентир для шапки;
- `логотип2.png` — как основа для будущего упрощённого знака;
- `review-1-redacted.png` и `review-2-redacted.png` — как источник для карточек отзывов;
- `permit-example-redacted.webp` — как источник для блока `пример документа`, после дополнительной обработки;
- `qualification-gip-redacted.webp` — как источник для блока доверия, только после подтверждения.

Пока не брать:

- `review-3-redacted.png` как сырой скриншот;
- любые документы из рабочих папок клиентов;
- любые фото объектов и договоры без отдельного отбора.

## 6. Что лучше сделать для современного сайта

Для нового сайта лучше не строить весь визуал на сырых скриншотах.

Правильная схема:

```text
реальный материал -> ручная проверка -> обезличивание -> дизайн-карточка -> вставка в сайт
```

Для визуального стиля сайта нужно подготовить:

- обновлённый спокойный wordmark на основе `Вектор Плюс-Про`;
- упрощённый знак для favicon;
- карточки отзывов в едином стиле;
- карточку документа без лишних данных;
- карточку квалификации/СРО;
- hero-визуал и схему пути клиента лучше сгенерировать/нарисовать отдельно, а не брать из старых сканов.
