## 3. Что именно сказал лектор по механике подписчиков

Смысл фрагмента из `Лекция_7.md`:

1. Найти конкурента с сильным каналом.
2. Посмотреть, где он покупал рекламу.
3. Через TGStat или похожую телега-аналитику увидеть список каналов, где были закупки.
4. Посмотреть, с какого канала конкуренту пришло больше всего подписчиков.
5. Перейти в этот канал и проверить, откуда уже этот канал сам покупал рекламу.
6. Отделить реальные сильные источники от накрутки и плохих ботов.
7. Размещать рекламу там, где аудитория похожа на целевую и есть признаки живого прироста.

Короткая формула:

```text
конкурент -> его рекламные закупки -> канал-донор подписчиков -> источники канала-донора -> проверка качества -> тестовое размещение -> UTM/CRM/ROMI
```

Это отличается от обычной идеи `давайте купим рекламу в канале`. Здесь сначала ищется цепочка, откуда реально пришли подписчики конкуренту.

## 4. Как это применить к ВПП

Для ВПП это нужно применить не как закупку любых подписчиков, а как поиск каналов, где есть люди, которые могут стать заказчиками или партнерами.

### 4.1. Кого анализировать

Первый список конкурентов и похожих источников:

- проектные бюро;
- архитектурные бюро;
- компании по согласованию перепланировок;
- проектирование коммерческих помещений;
- склады, ангары, реконструкция, fit-out;
- каналы про строительство бизнеса, коммерческую недвижимость, девелопмент, ТЦ, франшизы, ремонт бизнеса.

### 4.2. Что смотреть в TGStat/Telemetr

Для каждого канала:

| Поле | Зачем |
|---|---|
| Тематика канала | понять, есть ли там наша аудитория |
| Гео | Россия, платежеспособные города |
| Рост подписчиков | найти всплески после рекламы |
| ER / просмотры | понять, живой ли канал |
| Рекламные упоминания | увидеть, где канал покупал рекламу |
| Доноры подписчиков | найти источник прироста |
| Цена размещения | оценить тестовый бюджет |
| Комментарии / реакции | понять качество аудитории |
| Наличие ботов | отсечь плохие площадки |

### 4.3. Что считать успешным тестом

Не просто `прибавилось подписчиков`.

Для ВПП успешная закупка:

```text
новые подписчики -> вопросы -> присланные планы/ТЗ -> Bitrix24 сделки -> КП -> договор
```

Метрики:

- стоимость подписчика;
- стоимость вопроса;
- стоимость присланных исходных данных;
- стоимость квалифицированной сделки;
- конверсия из подписчика в обращение;
- конверсия из обращения в КП;
- ROMI по каналу.

## 5. Как встроить в нашу систему агентов

Новых агентов не создаем. Раскладываем по текущим 6.

| Роль | Что делает по этой механике |
|---|---|
| Agent 1 Scout | находит конкурентов и каналы, собирает watchlist |
| Agent 4 Publisher | готовит контент и рекламные креативы под найденные каналы |
| Agent 5 CRM | считает UTM, источник, подписчик -> заявка -> сделка -> деньги |
| Agent 6 Outreach | готовит аккуратные партнерские сообщения владельцам каналов, только после approval |

Agent 2 и Agent 3 подключаются, когда из канала пошли реальные обращения:

- Agent 2 фиксирует raw-сигнал;
- Agent 3 квалифицирует лид;
- Agent 5 заводит/обновляет сделку в Bitrix24.

## 6. Практический MVP для ВПП

Первый безопасный шаг без платных запусков:

1. Собрать 30 каналов-кандидатов:
   - 10 по коммерческой недвижимости;
   - 10 по строительству/ремонту бизнеса;
   - 5 по перепланировкам;
   - 5 по архитектуре/проектированию.
2. Для каждого вручную или через будущий read-only инструмент заполнить:
   - ссылка;
   - тематика;
   - подписчики;
   - просмотры;
   - ER;
   - признаки рекламных закупок;
   - подозрение на накрутку;
   - подходит / не подходит;
   - почему.
3. Выбрать 5 каналов для тестового размещения.
4. Под каждый канал сделать отдельный UTM:

```text
utm_source=telegram
utm_medium=channel_seed
utm_campaign=vpp_b2b_replan_or_warehouse
utm_content=<channel_id>
```

5. Вести переход не на абстрактный пост, а на конкретную страницу/сценарий:
   - коммерческая перепланировка;
   - склады/ангары;
   - реконструкция;
   - проверка исходных данных для КП.
6. Все обращения фиксировать в Bitrix24 с источником канала.

## 8. Что нельзя делать

- Не покупать рекламу только потому, что у канала много подписчиков.
- Не считать подписчиков результатом без вопросов и заявок.
- Не копировать чужие тексты и обещания.
- Не запускать массовые размещения без UTM и Bitrix24.
- Не покупать канал с подозрительной динамикой, даже если он дешевый.
- Не обещать ВПП `8000 подписчиков` как гарантированный результат.

## 9. Итог для ВПП

Лекторская механика применима так:

```text
TGStat/Telemetr разведка
-> найти каналы, откуда конкуренты реально получили подписчиков
-> проверить качество аудитории
-> сделать узкий оффер ВПП
-> дать полезный пост/разбор
-> перевести в MAX/сайт/лендинг
-> собрать исходные данные
-> завести Bitrix24 сделку
-> посчитать не подписчиков, а путь до КП и договора
```

Это нужно использовать как отдельный источник роста: `telegram_seed_channels`.
