# B2B Commercial Growth Sprint Plan

Дата: 2026-05-16

Статус: адаптированный план. Реклама, Bitrix24-запись, MAX, Telegram, VK, Яндекс Директ, тендерные площадки, Redis, scheduler, LLM, deploy и платные сервисы не запускались.

## 1. Что пришло как новая информация

Приоритет Яники:

```text
платежеспособные клиенты
-> преимущественно коммерция и юрлица
-> проектирование, ПД/РД, склады, ангары, ТЦ, магазины, офисы/fit-out
-> коммерческие перепланировки
-> реконструкция
-> вся Россия удалённо
-> есть СРО, опыт 10+ лет, безналичная оплата на расчётный счёт
-> заказов можно брать много
```

Дополнительное уточнение:

```text
отдельные лендинги будут нужны;
КП за 24 часа готовы делать при наличии полных данных от заказчика и автоматизации.
```

## 2. Как это применяем в нашем проекте

Мы не копируем чужую схему `Operating center · 24 agents`. У нас остаются 6 агентов.

Новая информация превращается в B2B growth-контур:

```text
Agent 1 Scout
-> спрос, города, конкуренты, тендеры, ABM-сегменты

Agent 4 Publisher
-> отдельные лендинги, офферы, кейсы, MAX/VK/Telegram/Дзен-контент

Agent 5 CRM
-> сделки Bitrix24, источники, UTM, сегменты, SLA, ROMI

AI-менеджер
-> первый ответ, сбор исходных данных, карточка сделки, передача человеку
```

Agent 2 и Agent 6 подключаются позже:

- Agent 2 — сбор тендеров, Avito/Profi/Яндекс Услуги, источники заявок;
- Agent 6 — ABM/аутрич только после approval и лимитов.

## 3. Сегменты запуска

| Приоритет | Сегмент | Что продаём | Комментарий |
|---:|---|---|---|
| 1 | Склады / ангары / производственные помещения | ПД/РД, конструктив, разделы под строительство или реконструкцию | Самый B2B-сегмент, хороший чек, нужна понятная ставка за м2 или диапазон. |
| 2 | ТЦ / магазины / ритейл / fit-out | проект перепланировки, инженерка, разделы для согласования с ТЦ/арендодателем | Нужны быстрые КП, перечень исходных данных и кейсы без раскрытия адресов. |
| 3 | Реконструкция коммерческих объектов | обследование, обмеры, проект реконструкции, экспертиза при необходимости | Сложнее, но выше ценность; AI должен быстро передавать человеку. |
| 4 | Перепланировка для юрлиц | комплект проектной документации, исходные данные, согласовательный путь | Не обещать согласование; говорить “подготовим проект и безопасный следующий шаг”. |
| 5 | Частные дома | эскиз/проект/КР/АР/разделы | Можно держать как второй поток, но не главный старт платного B2B-теста. |

## 4. Офферы, которые можно использовать

### Разрешённые формулировки

| Сегмент | Оффер |
|---|---|
| Склады / ангары | `ПД/РД для склада или ангара: состав работ, сроки и ставка после исходных данных` |
| ТЦ / магазины / fit-out | `Проект для коммерческого помещения: перепланировка, инженерные разделы, пакет для арендодателя или согласования` |
| Реконструкция | `Первичный разбор объекта: что нужно для проекта реконструкции, обследования и экспертизы` |
| Юрлица / коммерческая перепланировка | `Комплект проектной документации для коммерческого помещения после проверки исходных данных` |
| Быстрый вход | `КП за 24 часа при полном комплекте исходных данных` |
| Быстрый вход | `Предварительный состав работ по ТЗ за 24-48 часов` |

### Что можно писать только с условием

| Формулировка | Условие |
|---|---|
| `КП за 24 часа` | Только если заказчик прислал полный набор исходных данных: объект, город, площадь, задача, имеющиеся документы, сроки, контакт. |
| `ставка за м2` | Только после утверждения реальных диапазонов цен. Пока писать “ориентир после исходных данных”. |
| `экспертиза при необходимости` | Нормально, если не обещать прохождение экспертизы. |
| `авторский надзор` | Нормально, если услуга реально доступна для нужного типа объекта. |

### Что не используем

| Формулировка | Почему |
|---|---|
| `гарантируем согласование` | Нельзя обещать решение органа или экспертизы. |
| `согласуем любой объект` | Юридически и технически опасно. |
| `проект от X ₽/м2` | Нельзя без утверждённой цены и условий. |
| `3D-виз в подарок` | Не основной оффер для проектной организации; может привлекать не тех клиентов. |
| `лид-магнит в обмен на контакт` | Против правил проекта: не делаем обязательный контакт ради чек-листа. |

## 8. Как должен выходить результат

Не `лиды вообще`, а управляемая воронка:

```text
источник
-> посадочная страница / пост / карточка / тендер / ABM
-> входящее обращение
-> AI-менеджер собирает факты
-> сделка Bitrix24
-> ответственный и следующий шаг
-> КП / встреча / отказ / отложено
-> отчет по источнику и ROMI
```

В Bitrix24 для каждого обращения нужны:

- `SOURCE_ID`;
- `UF_CRM_VPP_SITE_SRC`;
- `UF_CRM_VPP_FLOW`;
- `UF_CRM_VPP_SCORE`;
- `UF_CRM_VPP_OBJ_TYPE`;
- `UF_CRM_VPP_AREA_M2`;
- `UF_CRM_VPP_FIRST_CH`;
- `UF_CRM_VPP_LAND_URL`;
- `UF_CRM_VPP_AI_STATUS`;
- комментарий с кратким резюме и недостающими фактами.

## 10. KPI как ориентир, не обещание

Используем только как гипотезу до первых данных:

| Сегмент | CPL-гипотеза | Лид -> встреча | Встреча -> договор | Цикл |
|---|---:|---:|---:|---|
| Частные дома / перепланировка | 800-2500 ₽ | 20-40% | 5-15% | короче |
| Коммерция / юрлица | 3000-8000 ₽ | 30-50% | 5-10% | 1-3 месяца |

В dashboard это должно быть помечено как `hypothesis`, пока нет реальных данных.

## 11. Что нужно от Яники

Не доступы в первую очередь, а решения:

1. Утвердить приоритет сегментов 1-5.
2. Утвердить города первой проверки.
3. Дать минимальные реальные ценовые ориентиры или запретить цены в рекламе.
4. Подтвердить формулировку `КП за 24 часа при полном комплекте исходных данных`.
5. Дать 3-5 обезличенных кейсов: тип объекта, м2, город, срок, что сделали.
6. Подтвердить, какие тендерные площадки уже есть или готовы подключать.
7. Подтвердить, кто отвечает на первые B2B-заявки в Bitrix24.
