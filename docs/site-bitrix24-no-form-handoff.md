# Связь сайта с Bitrix24 без формы сбора лидов

Дата: 2026-05-15.

Статус: архитектурное правило. Код сайта, Bitrix24, Open Lines, формы, CRM-обновления, внешние API и публикации не запускались.

## 1. Главный вывод

Сайт можно связать с Bitrix24 без формы на сайте.

Правильная схема для Вектор Плюс-Про:

```text
сайт
-> понятный CTA без формы
-> человек сам выбирает канал связи
-> MAX / Telegram / email / телефон
-> Bitrix24 Open Lines, телефония, ручная фиксация или Agent 5
-> сделка / задача / комментарий в CRM
```

Сайт не должен сам собирать персональные данные через форму. Он должен объяснять, что написать, и вести человека в канал, где он сам начинает обращение.

## 2. Что запрещено

Не делать на сайте:

- форму `имя + телефон + email`;
- квиз с обязательным контактом;
- лид-магнит в обмен на телефон/email;
- скрытую отправку данных в Bitrix24;
- автоматическое создание сделки без явного обращения человека;
- Bitrix24-виджет, если он работает как форма захвата;
- обещания `100% согласуем`, `согласуем всё`, `быстро без документов`.

Причина простая: в текущей стратегии сайт должен быть точкой доверия и понятного первого шага, а не агрессивной ловушкой контактов.

## 3. Что разрешено

Разрешено:

- кнопки `tel:` для звонка;
- кнопки `mailto:` для письма;
- ссылки на MAX-аккаунт;
- ссылки на MAX-канал `СИЛА Проекта`;
- ссылки на Telegram;
- ссылки на Яндекс Карты и 2GIS;
- текстовая подсказка, что написать в первом обращении;
- будущая обработка входящих сообщений через Bitrix24 Open Lines;
- ручная фиксация заявки менеджером в Bitrix24;
- будущая обработка входящего диалога Agent 5, если источник уже разрешён и проверен.

## 4. Как человек должен обращаться

На сайте нужно вести к такому действию:

```text
Напишите город, тип объекта, задачу и какие документы уже есть.
```

Пример первого сообщения клиента:

```text
Здравствуйте. Объект в Краснодаре, нежилое помещение 180 м2.
Нужен проект для реконструкции. Есть старый план и фото.
Хотим понять, какие документы нужны и с чего начать.
```

Это не форма. Это свободное сообщение в выбранном человеком канале.

## 5. Карта каналов

| Канал на сайте | Что делает человек | Как попадает в Bitrix24 | Источник в CRM |
|---|---|---|---|
| Телефон | нажимает и звонит | телефония Bitrix24 или ручная фиксация менеджером | `site_phone` |
| Email | пишет письмо | почта / ручная фиксация / будущий email intake | `site_email` |
| MAX-аккаунт | пишет сообщение | если подключено к Open Lines — автоматически; иначе вручную | `site_max` |
| Telegram | пишет сообщение | если подключено к Open Lines/боту — автоматически; иначе вручную | `site_telegram` |
| Яндекс Карты | звонит/пишет из карточки | через источник карт или ручную фиксацию | `yandex_maps` |
| 2GIS | звонит/пишет из карточки | через источник карт или ручную фиксацию | `2gis` |
| MAX-канал `СИЛА Проекта` | читает и переходит в контакт | через выбранный канал связи | `max_channel` |

Важно: если человек пришёл с сайта в MAX, источник лучше хранить не просто `MAX`, а `site -> MAX`.

## 6. Минимальная аналитика без формы

Даже без формы можно считать источник.

На сайте можно использовать безопасные ссылки с UTM-метками, если они не содержат персональные данные:

```text
?utm_source=site&utm_medium=contact_button&utm_campaign=main_contacts
```

Для мессенджеров можно использовать разные ссылки или разные тексты входа:

```text
MAX с главной: site_main_max
Telegram с контактов: site_contacts_telegram
Email со страницы ПД: site_service_pd_email
```

Но не добавлять в ссылку:

- имя человека;
- телефон;
- email;
- адрес объекта;
- кадастровый номер;
- документы;
- служебные данные.

## 7. Как Agent 5 должен видеть такое обращение позже

Когда обращение уже пришло в разрешённый канал, Agent 5 может создать или обновить сделку в CRM.

Минимальная карточка сделки для Bitrix24:

```text
source_id: CALL / EMAIL / 12|MAX_NUMBER_REDHAM / 2|MAX / 6|TELEGRAM_UNLIM / 2|TELEGRAM / WEB
site_source: site_max / site_telegram / site_email / site_phone / site_yandex_maps / site_2gis
first_touch_channel: сайт
contact_channel: MAX / Telegram / email / phone
city:
object_type:
task_type:
area_m2:
existing_docs:
raw_dialog_summary:
missing_facts:
recommended_next_step:
consent_status: explicit_message / unknown
landing_page: /
service_page:
```

Связь с уже зафиксированным intake-сценарием:

```text
сайт без формы
-> сообщение в MAX/Telegram/email/телефон
-> intake card
-> Agent 3 Processor
-> Agent 5 CRM
-> Bitrix24 deal
```

## 8. MVP без подключения форм

Первый безопасный MVP:

1. На сайте оставить кликабельные контакты.
2. В CTA ясно писать, что нужно отправить: город, тип объекта, задача, документы.
3. В Bitrix24 вручную или через Open Lines создавать/обновлять сделку и фиксировать сайтовую метку:
   - `site_max`;
   - `site_telegram`;
   - `site_email`;
   - `site_phone`;
   - `yandex_maps`;
   - `2gis`.
4. Первые обращения разбирать вручную.
5. После 10-20 обращений решить, какие каналы стоит автоматизировать.

Так мы не нарушаем запрет на формы, но всё равно строим CRM-учёт.

## 9. Что проверить перед автоматизацией

Перед любым подключением Bitrix24/Open Lines нужно отдельно проверить:

- какой канал реально подключён к Bitrix24;
- есть ли права на чтение/создание сделок;
- какие поля источника доступны;
- можно ли отличать `site -> MAX` от обычного `MAX`;
- не раскрываются ли webhook, токены и персональные данные;
- не создаются ли дубли контактов;
- что происходит, если человек написал без телефона;
- кто вручную проверяет первые обращения.

## 10. Что можно сделать на сайте позже

Позже, отдельной задачей и после подтверждения, можно:

- добавить разные `mailto:` темы для страниц услуг;
- добавить безопасные UTM к ссылкам MAX/Telegram, если каналы их сохраняют;
- добавить текст `Сообщите код: SITE-PD`, если нужно вручную отличать страницу услуги;
- добавить отдельные CTA на статических страницах услуг;
- подготовить список источников для Bitrix24;
- сделать read-only отчёт по входящим источникам.

Не делать это автоматически без проверки Bitrix24 и каналов.

## 11. Следующий маленький шаг

Не кодить CRM сразу.

Сначала подготовить таблицу:

```text
канал сайта -> ссылка/кнопка -> какой источник ставим в Bitrix24 -> кто обрабатывает -> как проверяем первые 10 обращений
```

Таблица подготовлена отдельно:

```text
docs/site-bitrix24-channel-routing-table.md
```

После этого можно точечно обновить сайт: не формой, а безопасными CTA и понятными источниками.
