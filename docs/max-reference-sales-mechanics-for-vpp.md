## 2. Главный вывод

Люди легче оставляют заявку, когда сначала видят маленькую пользу по своей ситуации.

В проекте MAX это выглядело так:

```text
пост
-> маленький AI-помощник / микро-разбор
-> понятный следующий шаг
-> аудит / внедрение / продукт
```

Для ВПП это должно стать так:

```text
пост / лендинг / MAX / карта / тендер
-> первичный инженерный разбор
-> сбор исходных данных
-> КП за 24 часа, если данных достаточно
-> сделка Bitrix24
-> проект / консультация / отказ / отложено
```

Простыми словами: продаем не “агентов”, а первый понятный шаг к проекту.

## 3. Что переносим в нашу систему

| Механика из MAX | Как адаптируем для ВПП |
|---|---|
| Маленький AI-помощник | AI-помощник первичного разбора объекта |
| Микро-разбор | Проверка ТЗ, плана, БТИ, площади, стадии и недостающих данных |
| Воронка доверия | Контент, кейсы, разборы ошибок, затем обращение |
| Offer readiness | Не зовем в то, что не можем выполнить сейчас |
| Один главный CTA | В каждом посте или лендинге один следующий шаг |
| Сбор вопросов аудитории | Фиксируем, что чаще спрашивают клиенты перед проектом |
| Лестница продуктов | От бесплатного первичного разбора к проектному договору |

## 4. Что не переносим

Не переносим:

- обещания заработка;
- агрессивную срочность “только 24 часа” без реальной причины;
- инфобизнес-слова вроде “миллионные воронки” в клиентских текстах;
- обещание результата, который зависит от экспертизы, органов согласования или третьих лиц;
- закрытый канал как обязательный вход;
- обязательный лид-магнит за телефон или email;
- массовые рассылки без approval.

## 5. Новый продуктовый язык для ВПП

Клиенту не нужно объяснять “многоагентную систему”.

Клиенту нужно объяснить:

```text
Мы быстро смотрим вашу задачу и говорим, какой проект нужен,
каких данных не хватает и какой следующий шаг безопаснее.
```

Внутренне это могут делать агенты, Bitrix24 и AI-менеджер.

Снаружи это должно звучать спокойно:

- `первичный разбор задачи`;
- `проверка исходных данных для КП`;
- `предварительный состав работ`;
- `инженерный маршрут проекта`;
- `КП за 24 часа при полном комплекте данных`.

## 6. Лестница доверия и продаж для ВПП

| Уровень | Что даем | Цель |
|---:|---|---|
| 0 | Посты, кейсы, разборы ошибок, карточки услуг | доверие |
| 1 | Бесплатный первичный разбор одного объекта по описанию | получить живую задачу |
| 2 | Сбор исходных данных для КП | понять, можно ли считать предложение |
| 3 | КП за 24 часа при полном комплекте данных | перевести интерес в сделку |
| 4 | Платная консультация / предпроектный разбор, если задача сложная | не тратить бесплатно часы на сложный объект |
| 5 | Договор на проектирование | основной доход |
| 6 | Авторский надзор / сопровождение / повторные проекты | допродажи и повторные сделки |
| 7 | Партнеры и B2B-пакеты | регулярный поток от УК, девелоперов, ритейла |

Уровень 4 нужен не для всех. Его можно включать только после утверждения Яникой.

## 7. Как это внедрить в текущих 6 агентов

| Агент | Что добавить |
|---|---|
| Agent 1 Scout | собирать боли и формулировки рынка: что люди спрашивают перед проектом |
| Agent 2 Collector | помечать источник и тип входа: тендер, сайт, карта, MAX, Avito, старая база |
| Agent 3 Processor | определять `segment`, `client_type`, `data_completeness`, `next_action` |
| Agent 4 Publisher | строить посты и лендинги вокруг микро-разбора, а не вокруг “мы лучшие” |
| Agent 5 CRM | заводить сделку с полями ВПП, статусом AI и недостающими данными |
| Agent 6 Outreach | давать не продажный спам, а точечный полезный ответ и предложение разобрать один шаг |

## 8. Главные AI-помощники для ВПП

Это не отдельные агенты. Это клиентские сценарии внутри системы.

| AI-помощник | Что делает | Для кого |
|---|---|---|
| `Помощник проверки исходных данных` | смотрит, хватает ли данных для КП за 24 часа | все сегменты |
| `Помощник перепланировки` | уточняет план, БТИ, задачу, тип помещения, ограничения | квартиры и коммерческие помещения |
| `Помощник коммерческого объекта` | уточняет объект, площадь, стадию, ПД/РД, экспертизу, сроки | юрлица |
| `Помощник реконструкции` | отделяет простую задачу от сложного объекта, где нужен человек | реконструкция |
| `Помощник тендерного запроса` | вытаскивает сроки, состав работ, требования и риски из ТЗ | тендеры |

## 9. Как должен выглядеть первый ответ AI-менеджера

Шаблон смысла:

```text
Поняла задачу. Чтобы понять состав проекта и можно ли подготовить КП за 24 часа,
нужно уточнить несколько вещей:

1. Тип объекта и город.
2. Площадь.
3. Что нужно: перепланировка, ПД/РД, реконструкция, разделы, экспертиза.
4. Какие документы уже есть.
5. Сроки и кто заказчик: физлицо, ИП, ООО, УК, девелопер.

Если пришлете имеющиеся планы, ТЗ или описание, мы соберем первичный состав работ
и передадим инженеру для проверки.
```

Важно: без обещания цены, согласования или экспертизы до проверки данных.

## 10. Что нужно внедрить первым

1. Полная offer and landing matrix по всем направлениям.
2. Поля `segment`, `client_type`, `data_completeness`, `next_action` в локальной логике Agent 3/Agent 5.
3. Контентная цепочка `пост -> микро-разбор -> исходные данные -> Bitrix`.
4. Первый AI-manager dry-run на 3 сценариях:
   - склад / ангар;
   - коммерческая перепланировка;
   - перепланировка квартиры.
5. Dashboard-блок: сколько обращений дошло до полного комплекта данных для КП.
