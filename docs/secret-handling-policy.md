# Политика безопасности ключей и токенов

Дата: 2026-05-06

Статус: обязательное правило проекта.

## Главное правило

Действует общий протокол:

```text
../PROJECT_SECRET_RULES.md
```

Если Яника намеренно передает токен, API-ключ, пароль приложения, webhook-ссылку или другой секрет для локальной настройки проекта, агент принимает его спокойно.

Агент обязан:

- не повторять секрет в ответе;
- не записывать секрет в `REPORT.md`, `README.md`, `CLAUDE.md`, `AGENTS.md`, `tasks/session-notes.md`, промпты, логи, скриншоты или публичные файлы;
- сохранять секрет только локально: `.env`, `.env.local`, gitignored `secrets/`, `~/.config/yanika-secrets/` или macOS Keychain;
- перед сохранением проверить `.gitignore`;
- при GitHub, деплое, архиве или передаче другому агенту заменять секреты на placeholders.

## Почему так

Чат не должен становиться местом хранения секретов. Даже если секрет передан намеренно для настройки, он должен сразу попасть только в локальное безопасное хранилище проекта.

Безопасная схема:

```text
секрет передан или введен локально -> попадает только в .env / secrets / Keychain -> не коммитится -> ассистент видит только SET/EMPTY
```

## Что нельзя делать с секретами

- повторять их в ответе;
- сохранять их в отчеты и документацию;
- писать их в логи;
- добавлять их в GitHub;
- вставлять их в публичные скриншоты;
- передавать другому агенту в открытом виде.

## Что можно писать в чат

- Название переменной, например `TELEGRAM_API_HASH`.
- Статус `SET`, `EMPTY`, `MISSING`.
- Ошибку без секрета.
- Скрин без видимых ключей.
- Вопрос, где взять ключ.
- Сам секрет, если Яника явно передает его для локальной настройки. В этом случае агент не повторяет значение и сразу сохраняет его только локально.

## Как безопасно заполнить секрет

Вариант 1. Вручную открыть `.env` на своём Mac и вставить значение самой.

Вариант 2. Использовать локальный скрипт:

```bash
.venv/bin/python scripts/set_env_secret.py TELEGRAM_API_HASH
```

Скрипт попросит значение в терминале без отображения текста. После ввода он запишет значение в `.env` и покажет только статус.

Важно: запускать скрипт нужно в обычном терминале на Mac. Не вставлять секрет в сообщение чата и не передавать его через команду вида `echo secret | ...`.

## Как ассистент проверяет секреты

Только так:

```text
TELEGRAM_API_HASH=SET
```

или:

```text
TELEGRAM_API_HASH=EMPTY
```

Ассистент не должен печатать реальные значения `.env`.

## Что делать, если секрет попал наружу

Если секрет случайно попал в GitHub, внешний сервис, общий документ, лог, скриншот, публичный файл или чужой чат:

1. Остановиться.
2. Предупредить Янику.
3. Рекомендовать перевыпуск.
4. Помочь удалить секрет из файлов, логов и истории, если это возможно.

## Как получить `TELEGRAM_API_HASH`

1. Открыть `https://my.telegram.org/apps`.
2. Войти по номеру телефона Telegram.
3. Открыть своё приложение.
4. Скопировать `api_hash`.
5. Не отправлять его в чат.
6. Записать локально:

```bash
.venv/bin/python scripts/set_env_secret.py TELEGRAM_API_HASH
```

После этого попросить:

```text
Проверь TELEGRAM_API_HASH безопасно, только SET/EMPTY.
```
