# MAX-публикации — короткая инструкция

Дата: 2026-05-06

Статус: кодовый адаптер подключён, реальная публикация ждёт `MAX_BOT_TOKEN` и `MAX_CHAT_ID`.

## Что уже сделано

- В Agent 4 есть команда публикации в MAX.
- Есть безопасный `dry-run`, который не вызывает MAX API.
- Реальная публикация идёт через официальный MAX Bot API: `POST /messages`.
- После успешной публикации система пытается записать событие `content_published` для Агента 5.

## Что нужно заполнить в `.env`

Не показывать реальные значения в чате и не коммитить их в GitHub.

```bash
MAX_BOT_TOKEN=
MAX_CHAT_ID=
MAX_API_BASE_URL=https://platform-api.max.ru
```

`MAX_BOT_TOKEN` берётся в MAX: Чат-боты -> Интеграция -> Получить токен.

`MAX_CHAT_ID` — ID чата или канала, куда бот имеет право отправлять сообщения. Если ID неизвестен, его можно найти через официальный метод `GET /chats`, когда бот уже добавлен в нужный чат/канал.

## Проверка без публикации

```bash
.venv/bin/python -m agents.agent4_publisher.cli publish max --file content/library/posts/draft-002-chto-bank-vidit.md --dry-run
```

Ожидаемый смысл результата:

```text
MAX API не вызван. Это безопасная проверка маршрута.
```

## Реальная публикация

Запускать только после того, как:

- `MAX_BOT_TOKEN=SET`;
- `MAX_CHAT_ID=SET`;
- выбран правильный файл поста;
- пост проверен глазами;
- понятно, что публикация нужна прямо сейчас.

Команда:

```bash
.venv/bin/python -m agents.agent4_publisher.cli publish max --file content/library/posts/draft-002-chto-bank-vidit.md
```

## После публикации

Если публикация прошла успешно, вручную добавить строку в:

```text
content/pipeline/published-log.md
```

Правило: не добавлять запись заранее. Только после фактической публикации.

## Важные ограничения

- MAX принимает текст до 4000 символов.
- Токен передаётся только через заголовок `Authorization`, не через query-параметр.
- Если Redis не работает, пост всё равно может быть опубликован, но событие для CRM не запишется. В этом случае не нажимать публикацию повторно, чтобы не создать дубль.
- Не публиковать один и тот же текст во все каналы без адаптации.
