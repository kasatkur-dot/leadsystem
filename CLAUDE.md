# Lead Intelligence Pipeline — ООО «Вектор Плюс-Про»

## Запуск

```bash
python -m orchestrator.scheduler        # запуск оркестратора
python -m pytest tests/                 # тесты
redis-cli ping                          # проверка Redis
```

Переменные окружения: `.env` (см. `.env.example`). Никогда не коммитить `.env`.

## Стек

Python 3.11+ · APScheduler · Redis (AOF включён) · PostgreSQL · Claude API (claude-sonnet-4-6) · Bitrix24 REST API · Telethon (user account) · python-telegram-bot · vk_api · MAX Bot API

## Два потока лидов

- **Поток А** — Перепланировки под ключ, вся Россия (B2C: физлица, покупатели, продавцы)
- **Поток Б** — Инженерное проектирование (КР/КЖ/КМ/АР/АС/ПОС, стадии П и Р), вся Россия (B2B: застройщики, архитекторы, ГИПы)

**Не берём:** технологические разделы, линейные объекты, одиночный раздел без комплекса.

## Контент-стратегия: 5 каналов, разный голос

Одна тема — разный угол. Контент между каналами НЕ дублируется.

| Канал | Аудитория | Поток | Голос | Уникальность |
|---|---|---|---|---|
| **MAX** «СИЛА Проекта» | B2C: покупатели/продавцы квартир, ремонтники | А | Бытовой, тёплый, про риски | Бот ПРОВЕРКА — интерактивный вход в диалог |
| **Telegram** | B2B: ГИПы, архитекторы, застройщики | А + Б (акцент Б) | Экспертный, инсайды отрасли | Авторский контент от Сергея (практикующий ГИП) |
| **VK** | B2C широкая: ремонт, недвижимость | А + жизнь компании | Разговорный, живой, с фото | До/после, команда, процесс работы |
| **Яндекс Дзен** | Поисковый трафик (оба потока) | А + Б | SEO-статьи, инструкции | Только длинный формат — ранжируется в Яндексе |
| **Яндекс Карты / 2GIS** | Локальный поиск Краснодар | А + Б | Новости, акции, фото | Привязка к геопозиции, отзывы |

### Пример: одна тема в 4 каналах (без дублирования)

Тема «ипотека и перепланировка»:
- MAX: «Банк требует документы — что делать» → боль + CTA: ПРОВЕРКА
- TG: «Почему банки ужесточили проверку в 2025 — разбор практика» → экспертно
- VK: «Клиент пришёл с отказом банка за 3 дня до сделки — вот как решили» → история
- Дзен: «Полное руководство: как согласовать перепланировку для ипотеки» → SEO 2000 слов

### Контент по Потоку Б (инженерное проектирование) — где публиковать

- **Telegram** — главная площадка: кейсы МКД/ТЦ/склады, разбор разделов, партнёрство
- **Дзен** — SEO по запросам «проект МКД», «субподряд проектирование», «рабочая документация»
- **VK** — изредка, кейсы крупных объектов как портфолио
- **TenChat** — B2B нетворкинг с застройщиками (Агент 6, Волна 2)
- **MAX** — минимально (аудитория не та)

## Shared-модули (shared/)

- `models.py` — RawLead, QualifiedLead, OutreachLead (Pydantic)
- `redis_client.py` — очереди raw/qualified/outreach + `blocklist` (Redis SET)
- `db.py` — PostgreSQL: LeadRecord, DuplicateFingerprint, AgentLog
- `logger.py` — loguru, единый логгер

## Критические правила

**Агент 6:** `approver` должен быть запущен всегда. Нет ответа 30 мин → лид не отправляется + алерт в TG.

**Claude API:** hard cap токенов в день в оркестраторе. 80% → алерт. 100% → стоп всех Claude-вызовов.

**Redis:** AOF persistence включён. Без него потеря лидов при перезапуске.

**Avito:** ротация User-Agent + задержки. 403/captcha → алерт + субагент стоп.

**Blocklist:** конкуренты/боты/спамеры → `shared/redis_client.py` blocklist. Фильтрация на входе в Агент 2.

**Контент:** разные каналы = разный угол подачи. Не копировать пост из MAX в TG без адаптации.

**Profi.ru:** подключение через Playwright с сохранённой сессией от рабочего аккаунта ВПП.
