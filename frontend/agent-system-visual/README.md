# Agent System Visual Frontend

Это рабочая копия фронтенда из исходной папки `Многоагентная система-визуал`.

Запускать через локальный backend:

```bash
.venv/bin/python -m backend.agent_dashboard_api.server --host 127.0.0.1 --port 8787
```

При запуске через backend файл `/data.js` генерируется из реального отчёта:

```text
data/reports/agent_dashboard.json
```

Исходная папка `Многоагентная система-визуал` не удалялась.

## Управление агентами

В нижней части интерфейса добавлен `Chat / voice command center`.

Он нужен, чтобы управлять агентами из диалогового окна и позже голосом:
- чат не перекрывает граф агентов;
- команды уходят в backend как локальный `task-handoff preview`;
- голосовая запись отправляется в dry-run endpoint, без реальной расшифровки;
- картинки и поиск по базе знаний пока только показывают выбранный маршрут модели.

Реальные внешние вызовы OpenRouter, Whisper, image-моделей и embeddings не запускаются без отдельного включения в `.env`.

Для локального теста до сервера использовать режим `LLM_RUNTIME_MODE=openrouter_free_test`: интерфейс будет отвечать через OpenRouter, но все текстовые LLM-задачи принудительно пойдут через одну бесплатную модель `LLM_MODEL_DEMO_FREE`.

Для показа мастеру на сервере использовать режим `LLM_RUNTIME_MODE=demo_server_free`: логика такая же, но настройки берутся из серверного окружения.
