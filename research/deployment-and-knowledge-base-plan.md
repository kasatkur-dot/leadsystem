## 3. Нужен ли Vercel этому проекту

Да, но не прямо для первого запуска Python-MVP.

Vercel нужен для:

| Что деплоить | Зачем |
|---|---|
| Мини-сайт студии | Доверие, формы, лид-магниты, SEO |
| Лендинги под каналы | Яндекс.Директ, VK Ads, Telegram Ads, Дзен/РСЯ |
| Лид-магнит ПРОВЕРКА | Чтобы человек мог пройти входную проверку по ссылке |
| Dashboard сквозной аналитики | Экран как у лектора: каналы, воронка, ROMI |
| Презентации / визуальные материалы | Отправлять клиентам и партнёрам ссылкой |

Не нужен прямо сейчас для:

```text
Gmail tender collector
Telegram monitoring
Redis
Bitrix24 routing
локальной проверки MVP
```

## 4. Когда подключать Vercel

Правильный порядок:

```text
1. Запустить базовый MVP: Gmail + Telegram + Bitrix24.
2. Добавить поля источника и аналитики в модели лида.
3. Создать простой лендинг или dashboard.
4. Подключить GitHub-бэкап.
5. Подключить Vercel к GitHub.
6. Деплоить сайт / dashboard по постоянной ссылке.
```

До шага 3 Vercel не должен тормозить запуск лидогенерации.

## 8. Whisper

Проверка показала: Whisper на компьютере действительно есть, но не в том виде, как обычная команда `whisper`.

Найдено:

```text
/Users/yanika/Library/Python/3.9/bin/mlx_whisper
/Users/yanika/Library/Python/3.9/lib/python/site-packages/mlx_whisper
/Users/yanika/.cache/whisper/tiny.pt
/Users/yanika/.cache/whisper/base.pt
```

Пакеты:

```text
mlx-whisper 0.4.3
mlx 0.29.3
```

Не найдено:

```text
обычная команда whisper
openai-whisper Python-модуль
ffmpeg / ffprobe
```

Что это значит:

```text
Whisper-следы есть.
mlx_whisper установлен.
Но для удобной работы с WebM-записями нужно доделать настройку.
```

Проблемы:

| Проблема | Что значит |
|---|---|
| `mlx_whisper` не в PATH | Его нужно запускать полным путём или добавить в PATH |
| Нет `ffmpeg` | WebM/видео может не обработаться корректно |
| `mlx_whisper` начал тянуть модель с HuggingFace | Нужна проверка модели и интернета |
| В `.cache/whisper` лежат `.pt` модели | Это похоже на модели OpenAI Whisper, а `mlx_whisper` использует MLX-модели |

Вывод: Whisper надо считать “частично установленным”.

Перед следующей расшифровкой видео нужно отдельным шагом:

```text
1. Проверить, можем ли добавить mlx_whisper в PATH.
2. Проверить или поставить ffmpeg.
3. Сделать короткий тест на маленьком аудио.
4. Только потом гонять длинные лекции.
```

## 9. Рабочая система лидов лектора как эталон

Скриншоты лекции показывают не просто список каналов.

Это эталон будущей рабочей системы:

```text
каналы -> расходы -> визиты -> лиды -> квалификация -> встречи -> сделки -> выручка -> прибыль -> ROMI
```

Что обязательно перенести в наш проект:

| Элемент у лектора | Что значит для нас |
|---|---|
| Выручка | Хранить сумму сделки из Bitrix24 |
| Прибыль | Считать хотя бы грубо: выручка минус расходы |
| ROMI | Видеть окупаемость каналов |
| CAC | Стоимость сделки / клиента |
| CR visit->deal | Конверсия от визита до сделки |
| Воронка | Визиты -> лиды -> квалиф. -> встречи -> сделки |
| Таблица каналов | Сравнение каналов по расходам и деньгам |
| Сортировка по ROMI | Быстро видеть лучшие и худшие каналы |
| Экспорт | Выгружать отчёт владельцу / менеджеру |
| Calltouch / CRM | Позже подумать о коллтрекинге и CRM-связке |
| amoCRM / Bitrix24 | У нас основной CRM-контур: Bitrix24 |

Это уже частично зафиксировано в:

```text
research/cross-channel-analytics-romi.md
```
