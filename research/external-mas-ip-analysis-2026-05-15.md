Дата: 2026-05-15

Статус: безопасный read-only анализ. POST-запросы в чат, аудио, логины, формы, сканирование портов, платные API и внешние действия не запускались.

## 1. Что это за система

По открытой странице это интерактивная карта многоагентной системы `МаС · Многоагентная система`.

Технически видно:

- сервер: `nginx/1.24.0 (Ubuntu)`;
- главная страница отдаётся по `GET /`;
- `HEAD /` возвращает `405 Method Not Allowed`, разрешён `GET`;
- фронтенд собран как статическая React-страница через CDN:
  - React 18.3.1;
  - ReactDOM 18.3.1;
  - Babel standalone 7.29.0;
- подключены модули:
  - `data.jsx`;
  - `nodes.jsx`;
  - `inspector.jsx`;
  - `app.jsx`;
  - `chat.jsx`;
  - `styles.css`;
- `robots.txt` и `sitemap.xml` возвращают `404`;
- чатовый слой обращается к same-origin API:
  - `POST /api/chat`;
  - `POST /api/chat/audio`;
- `GET /api/chat` возвращает `404`, POST не тестировался, чтобы не запускать возможный LLM/backend.

Вывод: это не просто картинка, а прототип операционного центра многоагентной системы с визуальной картой, сценариями и чатовым интерфейсом.

## 5. Что не стоит брать

| Элемент | Почему не берём сейчас |
|---|---|
| 27 ролей | Для нашего проекта это раздует систему и нарушит правило “не создавать новых агентов” |
| Робот-звонарь | Нужны отдельные согласия, записи, правила 152-ФЗ и бюджет |
| Платные рекламные роли | У нас сначала органика, CRM, Bitrix24 и один MVP-тест |
| Полный OSINT-арсенал | Может увести в сложную и юридически чувствительную зону |
| React + Babel прямо в браузере | Хорошо для прототипа, но не как production-архитектура |
| HTTP без домена и HTTPS | Нельзя использовать как публичный интерфейс с микрофоном и клиентскими данными |
| Чат POST без контроля стоимости | Может запускать LLM/backend, поэтому тестировать только после разрешения |

## 6. Риски и ограничения найденной системы

1. Сайт работает по `http://` на IP без домена и HTTPS.
2. Голосовой ввод через браузер на обычном HTTP-IP может не работать стабильно, потому что микрофон обычно требует защищённый контекст.
3. `robots.txt` и `sitemap.xml` не настроены, но для внутренней панели это не критично.
4. Фронтенд зависит от внешнего CDN `unpkg.com`; если CDN недоступен, интерфейс может не загрузиться.
5. API чата есть в коде, но backend не исследовался POST-запросами, чтобы не запускать возможные платные вызовы.
6. Данные агентов выглядят как демонстрационная спецификация, а не доказательство реального production-runtime.
