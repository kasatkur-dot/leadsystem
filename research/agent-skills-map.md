# Карта скиллов агентов

Дата: 2026-04-30
Проект: Lead Intelligence Pipeline, ООО «Вектор Плюс-Про»
Статус: решение из урока 3, перед техническим расширением агентов

## Главная мысль

В уроке была важная логика:

```text
агент без скиллов = роль, описанная текстом
агент со скиллами = роль + инструменты
агент со скиллами и субагентами = маленький отдел
```

Для нашего проекта это важно.

У нас уже есть 6 агентских отделов, но нужно явно зафиксировать, какие навыки и инструменты нужны каждому агенту.
Иначе агент будет “знать роль”, но не будет понимать, какими инструментами пользоваться.

## Решение по установке

Сейчас ничего не устанавливаем.

Причина:

```text
сначала нужно стабилизировать MVP и поля источника лида,
а потом усиливать конкретного агента конкретным скиллом.
```

Если ставить всё подряд, система станет тяжелее, но не умнее.

## Рекомендация по очередности скиллов

1. Для MVP не ставить новые скиллы, а завершить поля источника и сквозной аналитики.
2. Первым кандидатом на установку позже рассмотреть `web-scraping` для Агента 2, если будем собирать данные с открытых площадок.
3. Вторым кандидатом рассмотреть `telegram-bot` для Агента 5/6, если текущих Telegram-скриптов станет мало.
4. Третьим кандидатом рассмотреть `build-dashboard`, когда начнём делать экран `канал -> лид -> сделка -> выручка -> ROMI`.
5. Vercel-skill подключать только когда появится сайт, лендинг или dashboard, который нужно открыть по ссылке.

Пример запроса:

```text
Используй find-skills.
Найди полезные скиллы для Агента 5 CRM и сквозной аналитики:
Bitrix24, CRM reporting, dashboards, marketing attribution, spreadsheets.
Сначала покажи список, ничего не устанавливай.
```

## Агент 1 — Скаут / разведка

Роль: изучает рынок, конкурентов, офферы, цены, отзывы и тендерные площадки.

Нужные скиллы:

| Скилл | Зачем нужен |
|---|---|
| Web research | Искать конкурентов, площадки, источники спроса |
| Competitive analysis | Сравнивать офферы, цены, тексты, слабые места |
| Review analysis | Разбирать отзывы конкурентов и боли клиентов |
| Tender intelligence | Смотреть тендерные площадки и требования |
| Spreadsheet analysis | Сводить конкурентов, цены и источники в таблицы |
| Report writing | Делать короткие выводы для Агента 4 и Агента 5 |

Примеры задач:

```text
Найди 10 конкурентов по перепланировкам и проектированию.
Сравни их офферы, цены, отзывы и каналы трафика.
Дай вывод, что Агенту 4 улучшить в контенте и объявлениях.
```

## Агент 2 — Коллектор / сбор лидов

Роль: собирает сырые лиды из источников и сохраняет источник первого касания.

Нужные скиллы:

| Web scraping | Собирать заявки/сигналы с площадок |
| Marketplace monitoring | Avito, Profi.ru, Яндекс Услуги, YouDo |
| Maps monitoring | Яндекс Карты и 2GIS |
| HH / vacancy research | Искать B2B-сигналы по вакансиям |
| Data normalization | Приводить разные источники к единой модели RawLead |
| Consent/source tagging | Сохранять источник, канал, согласие и первое касание |

Примеры задач:

```text
Проверь 10 сырых лидов из разных источников.
Для каждого заполни source, source_type, first_touch_channel, consent_status и next_action.
```

## Что делать сейчас

Не устанавливать всё подряд.

Сначала:

```text
1. Сохранить эту карту.
2. Для каждого агента выбрать 2-3 самых важных скилла.
3. Через find-skills искать кандидаты на установку.
4. Ничего не ставить без проверки пользы.
5. После установки фиксировать в REPORT.md.
```

## Первый приоритет скиллов

Для ближайшего MVP важнее всего:

```text
Агент 2: email parsing, source tagging
Агент 3: lead scoring, data cleaning
Агент 5: Bitrix24, reporting, marketing attribution
Агент 6: Telegram monitoring, approval workflow
```

Агент 1 и Агент 4 важны, но их можно усиливать после запуска базового сбора и CRM.

## Итог

Нам нужна не просто папка `agents/`.

Нам нужна связка:

```text
агент -> зона ответственности -> нужные скиллы -> тестовая задача -> отчёт
```

Это следующий слой зрелости многоагентной системы.
