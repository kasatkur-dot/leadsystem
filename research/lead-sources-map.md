## Что за студия

Проектная студия. Основные объекты: дома, квартиры, школы, детские сады, торговые центры, ангары.

**Основная услуга:** рабочая документация, конструктивные решения, эскизы, планировки, посадка на объект. Все разделы в комплексе — можем. Упор на конструктив + остальные разделы комплексом.

**Не берём:**
- технологические разделы
- линейные объекты (дороги, сети)
- отдельные разделы, если клиент хочет только один раздел без комплекса

**Целевой клиент:** застройщики, владельцы коммерческой недвижимости, частные заказчики домов, учреждения (школы, сады).

---

## ICP — Идеальный клиент (Ideal Customer Profile)

✅ Подходит:
- Застройщик / девелопер — нужна РД на жилой дом, ТЦ, ангар
- Частный заказчик — дом, квартира, нужна РД или конструктив
- Учреждение / муниципалитет — школа, сад, нужна проектная документация
- Бизнес — ТЦ, ангар, нужны все разделы или конструктив

⛔ Не подходит:
- Запрос только на технологический раздел
- Линейные объекты (дороги, трубопроводы, сети)
- Запрос на один раздел без комплекса (кроме конструктива отдельно — уточнить)

## Приоритетные сегменты (уточнено владельцем)

✅ **Запросы на конструктив** — любой объект (ТЦ, склад, магазин, школа, жилой дом, что угодно)  
✅ **Перепланировки квартир + согласование**  
✅ **Коммерческие здания/сооружения, которым нужно разрешение на строительство**  
✅ **Коммерческие тендеры** — в первую очередь (не государственные)

⛔ Государственные тендеры (ЕИС) — не в приоритете старта

---

## Как читать карту

- **✅ ФАКТ (владелец)** = сказал владелец проекта напрямую
- **✅ ФАКТ (исследование)** = подтверждено реальными открытыми источниками
- **⚠️ ГИПОТЕЗА** = логичное предположение, требует проверки
- **Тип:** горячий = ищет сейчас; тёплый = скоро понадобится; партнёрский = другая компания приводит клиента; холодный = нет явного сигнала
- **Волна:** 1 = запускаем первыми; 2 = вторая очередь; 3 = долгосрочно

## Разбивка на подпроекты

Делим не по инструментам, а по типу источника и типу сигнала.

### Проект 1. Карта лидов и ICP (главный управляющий)
Что делает:
- описывает идеального клиента (ICP)
- определяет сегменты
- задаёт скоринг
- хранит схему оценки лида

**Без этого всё остальное — мусоросборник.**

### Проект 2. Официальные и стабильные источники
Источники: Yandex Maps API, Yandex Organization Search API, 2GIS Places API, официальные каталоги  
Статус: безопасная база для старта, API подтверждены

### Проект 3. Тендерный мониторинг
Источники: ЕИС, коммерческие тендерные площадки, закупки девелоперов  
Отдельный поток — другие поля, другой цикл, другой оффер  
Статус: ЕИС как источник подтверждён, но API-реализация требует отдельной валидации

### Проект 4. Партнёрские лиды
Источники: дизайнеры, строители, брокеры, агентства, смежные подрядчики  
Это не просто лиды — это канал партнёрского трафика

### Проект 5. Мониторинг прямых запросов и чатов
Источники: Telegram, сообщества, объявления, посты, локальные запросы  
Самый шумный, но иногда самый быстрый канал

### Проект 6. Обогащение и маршрутизация (центральный движок)
Не источник — это слой, который:
- чистит и дедуплицирует
- скорит
- распределяет по типам (горячий / тёплый / партнёрский / холодный)
- отправляет в CRM / Telegram / задачи / автообработку

---

## Первая волна — рекомендованный старт

### Волна 1. Коммерческие помещения + открывающийся бизнес
Источники: Yandex/2GIS, сигналы открытия бизнеса, партнёры (дизайнеры/строители/брокеры)  
Почему: ближе к регулярному спросу, шире для студии проектирования, не только перепланировки

### Волна 2. Прямые запросы на проектирование / согласование / перепланировку
Источники: чаты, площадки, объявления, тендеры  
Почему: горячие лиды, но источник шумнее и сложнее

### Волна 3. Партнёрский поток
Источники: дизайнеры, строители, брокеры, агентства  
Почему: самые дешёвые и сильные лиды, но строятся дольше

## Целевая архитектура пайплайна

```
Источники → Сбор → Дедупликация → Обогащение → Скоринг → Маршрутизация → Действие
                                                     ↓
                                        горячий → продажи
                                        тёплый  → дообогащение → очередь
                                        партнёрский → партнёрский канал
                                        холодный → архив
                                                     ↓
                                        CRM / Telegram / таблица / задача / автообработка
```

Это **lead intelligence pipeline**, а не просто парсер.
