---

## Ключевой принцип

**Оркестратор управляет всем. Агенты работают независимо, параллельно.**

Один Python-процесс (оркестратор) запускает агентов по расписанию.
Данные между агентами — через Redis-очередь.
Все лиды проходят через центральный пайплайн.
На выходе — квалифицированная карточка в Bitrix24 + уведомление менеджеру.

**Стек:** Python + APScheduler + Redis + PostgreSQL + Claude API + Bitrix24.
**Без n8n и Make.**

**НЕ берём:** технологические разделы, линейные объекты (дороги, сети), одиночный раздел без комплекса.

## Центральный пайплайн

```
┌─────────────────────────────────────────────────────────────┐
│  АГЕНТЫ-ИСТОЧНИКИ (параллельно)                            │
│                                                             │
│  [2: Коллектор]           [6: Аутрич]                     │
│  Email │ Avito │ Profi    TG-чаты │ VK │ Форумы           │
│  ЯУ │ HH │ ЦИАН │ YouDo                                   │
└──────────────────────┬──────────────────────────────────────┘
                       │ сырые лиды → Redis
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  АГЕНТ 3: ПРОЦЕССОР                                         │
│                                                             │
│  1. Дедупликация                                           │
│  2. Фильтрация (наш профиль? нет технологии / линейных)   │
│  3. Определение потока (перепланировка vs проектирование)  │
│  4. Обогащение (компания, контакт, площадь, объект)        │
│  5. Скоринг: горячий / тёплый / холодный / нецелевой      │
│  6. Оффер: что предложить конкретно этому лиду             │
└──────────────────────┬──────────────────────────────────────┘
                       │ квалифицированный лид → Redis
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  АГЕНТ 5: CRM-РОУТЕР                                        │
│                                                             │
│  Горячий   → Bitrix24: сделка + «Позвонить сегодня»       │
│              + TG-уведомление 🔥                           │
│  Тёплый    → Bitrix24: лид + «Написать в течение дня»     │
│              + TG-уведомление ⚡                           │
│  Партнёр   → Bitrix24: партнёрская очередь + TG 🤝        │
│  Нецелевой → Архив                                         │
│                                                             │
│  При достаточных данных → KP-генератор → черновик КП      │
└─────────────────────────────────────────────────────────────┘
```

## Агент 6: Цикл одного аутрич-ответа

```
[Мониторинг TG/VK/форумов]
       │
       ▼
[Найден пост / комментарий]
       │
       ▼
[Claude API: оценить релевантность 0–10]
       │
     ≥ 7?
    /    \
  Да     Нет → пропустить
   │
   ▼
[Claude API: сгенерировать ответ — уникальный, в стиле человека]
   │
   ▼
[Telegram тебе: пост + предложенный ответ]
   │
  ✅         ✏️            ❌
   │          │             │
отправить  отредактировать  пропустить
   │
   ▼
[Задержка 2–15 мин + рандомизация]
   │
   ▼
[Отправка ответа]
   │
   ▼
[Мониторинг: ответил ли человек?]
   │
  Да → уведомление менеджеру → ручной диалог
  Да + интерес → карточка в Bitrix24 (через Агент 5)
```
