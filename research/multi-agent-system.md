# Мультиагентная система — Lead Intelligence Pipeline ВПП

Версия: 2.0
Дата: 2026-04-26
Продукт: автоматическая добыча, квалификация и вовлечение лидов для ООО «Вектор Плюс-Про»

---

## Принцип системы

**Оркестратор управляет всем.** Агенты работают параллельно, независимо, по расписанию.
Данные между агентами передаются через общую очередь (Redis).
Все лиды проходят через единый центральный пайплайн.
На выходе — квалифицированная карточка в Bitrix24 + уведомление менеджеру.

**Стек:** Python (asyncio) + APScheduler + Redis + PostgreSQL + Claude API + Bitrix24 REST API.
**Без n8n и Make** — всё собственное на Python.

## Агент 1. Разведчик рынка (Market Intelligence Agent)

**Роль:** аналитик. Следит за конкурентами, изучает их тексты, офферы, цены.

**Задачи:**
- Мониторит объявления конкурентов на Avito, Profi.ru, Яндекс Услугах, Яндекс Картах, 2GIS
- Сохраняет тексты, заголовки, цены, УТП конкурентов
- Анализирует: что пишут, какие боли закрывают, какие слова используют
- Сравнивает с нашим позиционированием, предлагает улучшения
- Отслеживает появление новых игроков на рынке

**Субагенты:**

| Субагент | Источник | Что собирает |
|---|---|---|
| Avito-разведчик | Avito | Тексты объявлений конкурентов, цены, рейтинги |
| Profi-разведчик | Profi.ru | Профили конкурентов, отзывы, цены |
| Карты-разведчик | Яндекс Карты, 2GIS | Карточки конкурентов, отзывы, рейтинги |
| Контент-разведчик | VK, Telegram, Max конкурентов | Тексты постов, частота, вовлечённость |
| Тендер-разведчик | Тендерные площадки | Кто участвует, по каким ценам выигрывает |

**Выход:** отчёт с инсайтами + рекомендации по текстам → Оркестратор → Агенту 4

## Агент 3. Процессор лидов (Lead Processor Agent)

**Роль:** квалификатор. Принимает сырые лиды, очищает, обогащает, оценивает.

**Шаги обработки:**

| Шаг | Что делает |
|---|---|
| Дедупликация | Убирает дубли по контакту, тексту, источнику |
| Фильтрация | Отсеивает нецелевые (технология, линейные объекты, одиночный раздел) |
| Определение потока | Перепланировка или инженерное проектирование? |
| Обогащение | Название компании, сайт, телефон, город, тип объекта, площадь |
| Скоринг | Горячий / тёплый / холодный / нецелевой |
| Оффер | Что конкретно предложить этому лиду |

**Логика скоринга:**

```
Горячий   = явный запрос + контакт + известен объект/площадь
Тёплый    = сигнал спроса + есть контакт, но запрос не оформлен
Холодный  = слабый сигнал, нет контакта или явного запроса
Нецелевой = не наш профиль (технология, линейные, одиночный раздел) → архив
```

**НЕ берём:**
- Технологические разделы
- Линейные объекты (дороги, сети)
- Одиночный раздел без комплекса

**Субагенты:**

| Субагент | Задача |
|---|---|
| Чистильщик | Дедупликация + базовая фильтрация |
| Обогатитель | Поиск дополнительных данных о компании/человеке через открытые источники |
| Скорер | Claude API — присваивает оценку и тип |
| Оффер-генератор | Claude API — формирует рекомендованный оффер под конкретный лид |

**Выход:** квалифицированная карточка лида → Redis-очередь → Агент 5

## Агент 5. CRM-роутер (CRM Router Agent)

**Роль:** финальная точка пайплайна. Получает квалифицированный лид и отправляет куда нужно.

**Задачи:**
- Создаёт карточку в Bitrix24 с правильным статусом и полями
- Назначает задачу менеджеру (позвонить / написать / обработать)
- Отправляет уведомление в Telegram с кратким описанием лида
- При достаточных данных — запускает KP-генератор → черновик КП менеджеру
- Ведёт статистику: откуда лиды, какая конверсия, какой поток эффективнее

**Логика маршрутизации:**

| Скор | Действие в Bitrix24 | Уведомление в TG |
|---|---|---|
| Горячий | Сделка «Новый» + задача «Позвонить сегодня» | 🔥 Горячий лид + описание |
| Тёплый | Лид «В работе» + задача «Написать в течение дня» | ⚡ Тёплый лид |
| Партнёрский | Контакт + задача «Партнёрская проработка» | 🤝 Партнёрский сигнал |
| Нецелевой | Архив | — |

**Субагенты:**

| Субагент | Задача |
|---|---|
| Bitrix-интегратор | Создание и обновление карточек в CRM |
| KP-триггер | Запуск KP-генератора при горячем лиде с достаточными данными |
| Нотификатор | Telegram-уведомления менеджеру |
| Аналитик | Статистика по источникам, потокам, конверсии — еженедельный отчёт |

## Агент 6. Социальный аутрич (Social Outreach Agent)

**Роль:** живое присутствие в сети. Находит людей с реальным интересом к нашим услугам и вступает в диалог как реальный человек — не как реклама, а как экспертная помощь.

**Охват:** вся Россия. Оба потока.

### Темы мониторинга

**Поток А — Перепланировки:**
> перепланировка, снос стены, объединение квартир, согласование, узаконить, самовол, проект квартиры, нежилое в жилое, перевод, кто делает перепланировку, посоветуйте по перепланировке

**Поток Б — Проектирование:**
> проект МКД, ТЦ, торговый центр, склад проект, ангар, производственное здание, рабочая документация, нужен проектировщик, субподряд проектирование, конструктив, КР КЖ КМ, госэкспертиза, BIM Revit, кто делает проект, посоветуйте проектировщика

|---|---|---|
| Telegram-чаты (ЖК, ремонт, строительство, Краснодар, города России) | Мониторинг сообщений + ответы | 1 |

### Система маскировки под человека

**Аккаунт-легенда:**
```
Имя:    реальное русское имя (Алексей / Дмитрий / Наталья)
Фото:   AI-сгенерированное реалистичное лицо
Bio:    «Строительство и проектирование»
Прогрев: 2–4 недели до аутрича — вступает в чаты, лайкает, иногда отвечает нейтрально
```

**Поведенческие паттерны:**
```
Время активности:  08:30 — 22:30 (не круглосуточно)
Обеденный перерыв: 13:00 — 14:30 (молчит)
Ночь:              не отвечает
Выходные:          активность -60%

Скорость «печати»: задержка 3–8 сек на каждые 100 символов
Задержка реакции:  2–15 минут после появления поста (не мгновенно)
Рандомизация:      ±30% к каждой задержке
Лимит откликов:    5–8 в день на аккаунт
Избирательность:   отвечает на 1 из 3–4 найденных постов (не на все подряд)
```

**Вариативность текстов:**
Claude API генерирует уникальный ответ каждый раз — никаких шаблонов.
Стили: экспертная помощь / уточняющий вопрос / краткий факт + приглашение.

**Пример цикла:**
```
Пост в TG-чате: «Купили квартиру в ЖК Радуга, хотим убрать стену между кухней и залом, с чего начинать?»

Claude оценивает релевантность: 9/10

Claude генерирует ответ:
«Начать нужно с техзаключения — нужно убедиться что стена не несущая.
 Без него БТИ не примет проект. Мы делаем под ключ: техзаключение + проект + согласование.
 Если интересно — напишите в личку, скажем сразу что реально в вашем случае»

→ Тебе в Telegram: [пост] [предложенный ответ] ✅ / ✏️ / ❌
→ После ✅: задержка 4–12 мин → отправка
→ Если ответили → уведомление менеджеру → ручной диалог
→ Заинтересованный → карточка в Bitrix24 (через Агент 5)
```

### Технические меры защиты

| Угроза | Решение |
|---|---|
| Блокировка по IP | Мобильный прокси с ротацией каждые 10–30 мин |
| Fingerprint устройства | Уникальный user-agent на аккаунт |
| Одинаковые тексты | Claude генерирует уникально каждый раз |
| Частые действия | Rate limiter + рандомные паузы |
| Новый аккаунт | Прогрев 2–4 недели перед началом аутрича |
| Telegram anti-spam | Telethon (user account, не bot token) — выглядит как человек |
| VK anti-spam | vk_api (user token, не группа) + лимиты |
