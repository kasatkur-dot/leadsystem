# Исследование каналов и конкурентов для контента

Дата: 2026-05-06

Статус: безопасная карта мониторинга. Массовый парсинг, Apify, PostMyPost, платные API и реальные публикации не запускались.

## 1. Главный вывод

Да, контент для студии нужно делать не только из внутренних идей.

Правильная схема такая:

```text
рынок и конкуренты -> паттерны тем -> вывод для нашей аудитории -> уникальный пост ВПП -> review -> публикация позже -> аналитика
```

Важно: мы не копируем чужие тексты, визуалы и офферы. Мы смотрим:

- какие боли повторяются;
- какие темы собирают внимание;
- какие CTA используют конкуренты;
- какие вопросы задаёт аудитория;
- где конкуренты обещают слишком много и как нам объяснить честнее;
- какие темы можно упаковать лучше и проще.

## 2. Где хранится watchlist

Файл:

```text
content/library/sources/competitor-channel-watchlist.csv
```

Он нужен как вход для Agent 1 Scout.

Agent 1 должен выдавать Agent 4 не просто ссылки, а готовые исследовательские карточки:

```text
source_id, тема, боль аудитории, доказательство, формат, CTA, риск копирования, наша уникальная подача
```

## 5. Как это должно входить в контент-план

В месячном плане для каждого поста должны быть поля:

```text
Research source
Audience pain
Competitor pattern
Our angle
Risk of copying
CTA
Status
```

Пример:

```text
source: mostpr.ru + Яндекс Недвижимость
pattern: конкуренты объясняют документы и сроки
our angle: чек-лист перед сделкой без запугивания
risk: не копировать структуру сайта и формулировки
CTA: задать вопрос в личные сообщения
```

## 6. Что мониторить еженедельно вручную

Для первого безопасного цикла не нужен Apify.

Достаточно раз в неделю вручную заполнить 5-10 строк:

| Поле | Что писать |
|---|---|
| `source_id` | Откуда взята идея |
| `date_seen` | Когда увидели |
| `topic` | О чём публикация или страница |
| `audience_pain` | Какую боль закрывает |
| `format` | пост, карточка, статья, кейс, FAQ, видео |
| `cta` | куда ведут человека |
| `engagement_signal` | просмотры, реакции, комментарии, видимый интерес |
| `our_angle` | как ВПП может сделать лучше/честнее |
| `risk_notes` | что нельзя копировать |
| `next_content_id` | к какой нашей идее привязать |

## 7. Что делать с каналами, которые знает Яника

Их нужно добавить в watchlist, но не в чат с секретами и не как хаотичный список.

Для этого создан шаблон:

```text
content/library/sources/yanika-known-channels-template.csv
```

Когда Яника пришлёт ссылки, мы не будем фантазировать и не будем анонимизировать смысл канала. Мы добавим их в `competitor-channel-watchlist.csv` с понятной пометкой:

```text
status = user_provided
owner_agent = agent1_scout
```

Готовый промпт для Яники:

```text
Отправь в Codex список каналов/сайтов конкурентов для анализа.
Можно просто ссылками, по одной в строке.
Если знаешь, подпиши рядом: конкурент, полезный канал, пример хорошего контента или источник идей.
Не отправляй логины, пароли, закрытые переписки и личные данные клиентов.
```

Куда отправить: сюда же, в Codex, в этом проекте.

## 8. Что не делаем сейчас

- Не запускаем массовый парсинг.
- Не скачиваем чужой контент пачками.
- Не копируем тексты и визуалы.
- Не используем закрытые переписки.
- Не подключаем Apify без бюджета и лимита.
- Не публикуем посты без review.
